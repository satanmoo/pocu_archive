## TSP-2 근사 알고리듬에서 사용하는 그래프

- 완전 그래프
    - 모든 정점 쌍이 연결되어 있음
    - 루프 없음
    - 변의 가중치는 음수가 아님
    - 변의 가중치는 삼각 부등식 만족

## 헤밀턴 경로

- 완전 그래프에서 모든 노드를 한 번씩만 방문하는 경로
- TSP-2 근사에서 이를 시용하는 이유는, 비용을 최소화 하기 위함

## 헤밀턴 순환

- 헤밀턴 경로에서 원래 출발 노드로 돌아오는 순환

## TSP-2 근사 알고리듬의 목표

- 헤밀턴 경로 중 가중치의 합이 최소인 최소 비용 경로를 찾는 것이 목표
  - 완전 그래프라서 MST를 구할 수 있고
  - 삼각 부등식을 만족하는 유클리드 TSP라서 성립함

## 유클리드 TSP

- 삼각 부등식을 만족하는 완전 그래프의 최소 비용 헤밀턴 순환을 구하는 문제
    - 루프 없음
    - 변의 가중치는 음수가 아님
- 이렇게 삼각 부등식을 만족해도 유클리드 TSP는 NP 완전 문제임
    - NP-hard에서 쉬워지긴 함..

## 유클리드 TSP에서 MST를 크루스컬 알고리듬으로 구할 때 반드시 MST를 찾을 수 있는 이유?

- 완전 그래프라서 어떤 노드에서도 다른 노드가 이어

## TSP-2근사 알고리듬 동작 절차

- MST 만듬
    - 시간 복잡도 O(ElogE)
    - 크루스컬 알고리듬 사용한다고 가정
- 시작점이 주어지고, 시작점에서 부터 MST DFS
    - 이 때 Double Walk로 구현
- DFS로 구한 순서대로 헤밀턴 순환에 포함
    - 이 때 중복 방문은 허용 X
    - 중복 방문을 스킵해도 되는 이유:
        - 완전 그래프라서!

- DFS 순서에 따라서 최소 비용 헤밀턴 순환은 여러 개 가능
