## DFS 코드

- 전위, 후위 구분

```java
import java.util.ArrayList;
import java.util.List;
import java.util.Stack;

public class Program {

    public static void main(String[] args) {

    }

    private static void searchDepthFirst(final Node node) {
        final Stack<Node> stack = new Stack<>();

        stack.push(node);
        while (!stack.isEmpty()) {
            Node current = stack.pop();

            System.out.println("전위");

            for (Node child : current.children) {
                stack.push(child);
            }

            System.out.println("후위");
        }
    }

    static class Node {
        int data;
        List<Node> children = new ArrayList<>();
    }
}
```

## BFS 코드

```java
package academy.pocu.comp3500.lab8.app;

import java.util.ArrayList;
import java.util.LinkedList;
import java.util.List;
import java.util.Queue;

public class Program {

    public static void main(String[] args) {

    }

    private static void searchDepthFirst(final Node node) {
        final LinkedList<Node> queue = new Queue<>();

        queue.push(node);
        while (!queue.isEmpty()) {
            Node current = queue.peek();

            System.out.println("전위");

            for (Node child : current.children) {
                queue.push(child);
            }
        }
    }

    static class Node {
        int data;
        List<Node> children = new ArrayList<>();
    }
}
```

## DFS vs BFS

- 트리의 형태(깊이가 깊냐, 너비가 넓냐)에 따라 메모리 사용량이 달라짐
    - 깊으면 DFS가 메모리 많이 사용
    - 넓으면 BFS가 메모리 많이 사용

- DFS의 경우 노드가 자식을 연속적으로 참조하기 때문에 캐시 메모리에 친화적

- 트리 DFS의 경우 추가적인 자료구조 없이 병렬 처리할 수 있음
    - 구현이 간단하다는 장점
    - 서브 트리는 서로 독립적이라 각각 스레드 풀에 제출하면 됨

- 트리(혹은 그래프) BFS 병렬 처리할 때 다음에 처리할 자식들을 관리하는 큐가 thread-safe 해야함
    - 오버헤드 발생
    - 복잡한 락 처리

- 일반적 그래프 DFS의 경우 순환 때문에 전역 방문 집합이 필요하고, 이 자료구조는 thread-safe 해야함
    - 오버헤드 발생
    - 트리는 순환이 없기 때문에 병렬 처리가 간단
