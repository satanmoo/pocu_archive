# Lab7

- 트라이 구현은 쉬웠음
- 트라이를 어떻게 활용하는가?
    - 어려움
- 암호문을 만들 때 부호어에서 문자들의 위치를 섞음
    - 따라서 입력 부호어에서 문자들의 위치는 의미가 없음
    - 입력 부호어를 "key는 문자, value로 문자의 개수" 이렇게 표현 가능
    - DFS 이용해서 트라이 검색하면서, 경로 복원하고, 와일드 카드 처리

- 문자를 소모하면서 탐색할 때 왜 와일드 카드보다 그냥 문자를 먼저 써야하는지 증명은 어렵지만, 반례를 만들기는 쉬움

- 삽질 많이 했지만, 아이디어가 재밌는 문제!
