## 기타 if 문 관련

![[Pasted image 20260206004916.png]]

> switch/case 문: 없음

본질적으로 *swithch/case 문* 은 if 문을 여러개 사용한 개념

> [!INFO] *swithch/case 문* 은 어떤 경우 컴파일러가 *Lookup table* 로 바꿀 수 있다.

> 조건 연산 (삼항 연산자) : 없음

본질적으로 *삼항 연산자* 는 if-else 문을 1줄로 만든 개념

> [!INFO] 삼항 연산자와 비슷한 동작을 하는 니모닉이 존재하는 CPU도 있기 때문에 C,C++ 개발자 중에서 삼항 연산자를 엄청 선호하는 사람도 있다. 삼항 연산자를 사용하면 컴파일러가 조건 연산 니모닉으로 바꾸는 최적화를 하기 때문이다.

> 논리 연산: and/ora/eor 이후 N, Z 플래그에 따라 분기

[[020. 논리 연산]] 에서 본 것 처럼 *N, Z 플래그* 가 변한다.

> N : 최상위 비트가 있는지만 확인 가능 (1비트 불 값)
> Z : 8 비트 불 값

[[008. 제로 플래그 Z, 음수 플래그 N]] 에서 본 내용이다.
 
## bit

![[Pasted image 20260206010550.png]]

> A 레지스터 값을 바꾸지 않는 and 연산

[[020. 논리 연산]] 의 *and, ora, eor* 과 다르게 *A 레지스터* 의 값을 바꾸지 않는다.

> Z 플래그에 결과가 저장됨
> Z == 1 : and 결과 모든 비트가 0

원래 [[008. 제로 플래그 Z, 음수 플래그 N]] 에서 본 *Z 플래그* 는 결과의 모든 비트가 0이면 1이 되었다. 마찬가지 논리가 적용된다.

모든 비트가 0을 만족하지 못하면 *Z 플래그* 는 0이 된다.

> M의 bit 7과 bit 6을 N, V 플래그에 대입

*M의 bit7 과 bit6* 에 따라 분기를 할 수 있다.
- [[003. 분기]], [[004. 고수준 조건문 옮기기]] 에 서본 니모닉에 응용 가능
