### CPU가 RAM에서 데이터를 읽을 때 사용하는 핀 위주로 타이밍 다이어그램 해석하기

![[Pasted image 20260101172129.png]]

"PHI2": 클럭, 내려가고 올라갈 때 한 사이클
- CPU는 사이클에 맞춰서 주소 핀을 세팅하고, 어느정도 시간이 흐르면 데이터를 읽어온다.
- CPU는 RAM에 신호를 보낸다. 이 신호는 사이클에 기반한다.
- RAM은 신호(주소값)에 따라 타이밍에 맞게 데이터를 출력한다.

![[Pasted image 20260101172424.png]]

내려오고, 올라가면 "1 사이클"
- 이 때문에 PHI"2"

CPU 스펙은 "1MHz"이다. 초당 100만 사이클이다.
- 한 사이클은 "1 μs"(1 마이크로 세컨드) 걸린다.
- 한 번 올라갈 때 "500 ns"
- [[007. 클럭 및 기타 선 연결하기#E) 클럭(발진기, 오실레이터)을 연결]] 참고

![[Pasted image 20260101174812.png]]

![[Pasted image 20260101174936.png]]

#### tAH

![[Pasted image 20260101174944.png]]

주소를 잡고 있는 시간

한 사이클이 끝나고, 주소를 얼마 시간 동안 홀드(출력)하는 개념이다.
- CPU 스펙에 따라 정의한다.
- "1000 ns" 다음 "tAH"동안 주소를 유지한다. (출력한다.)

이 주소는 왜 필요할까? CPU와 RAM이 주소를 기반으로 매핑되기 때문이다.

#### tADS

![[Pasted image 20260101175311.png]]

사이클이 시작되고 주소를 출력한다. 그리고 주소를 "Setup"하는데 시간이 걸린다.

이 시간이 지나야 주소핀(A0-A15)에서 유효한 주소값이 읽힌다.

초록색으로 하이라이트한 시간 동안은 유효한 주소가 유지된다.

![[Pasted image 20260101175559.png]]

주소가 유효하기 때문에 RAM에서 이 주소를 읽어서 사용해도 상관없다.

RAM에서 주소를 인지하고, 데이터핀을 통해 CPU에 데이터를 입력해준다.

CPU가 데이터핀을 통해 데이터를 읽을 때 RAM에서 데이터 핀으로 보내는 신호가 유효한 기간이 있다. 그 유효한 기간 동안에 CPU가 데이터를 읽어와야지 문제가 없다.

결론적으로 주소 세팅 => 데이터 읽기
