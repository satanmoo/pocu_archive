# zp를 이용한 반환값 전달

## 함수의 반환값

![[Pasted image 20260228230747.png]]

반환값의 전달 과정:
1. 함수 내부: `rts` 전에 반환값을 저장
2. 호출자: 그 값을 꺼내 사용

**호출자가 반환값을 언제나 확실히 찾을 수 있어야 한다.**
- *그 위치가 정해져 있어야 하며, 여러 가지 방법이 있다 — 호출 규약.*


## 반환값을 전달하는 다양한 방법

![[Pasted image 20260228230908.png]]

1. **페이지 0**을 통해
2. **스택**을 통해
3. **A, X, Y 레지스터**를 통해

> [!INFO] 고수준 언어와 함수의 반환값
> 
> *참고로 고수준 언어는 레지스터를 사용해서 반환하는 경우가 많다. 레지스터로 반환하는 것이 가장 빠르기 때문이다.*


## 방법 1-1: 함수마다 다른 페이지 0 주소 사용

![[Pasted image 20260228231147.png]]

> `(0x01,0x02) -> 0x00 | <A,P>`
> `(0x11,0x12) -> 0x10 | <A,P>`

주석에서 반환값이 어느 주소에 저장되는지 알 수 있다.
- `add` 함수의 경우 `$00`
- `sub` 함수의 경우 `$10`

![[Pasted image 20260228231458.png]]

각 함수에서 반환하기 전 `sta`로 **지정된 주소에 반환값을 저장**한다.

서로 **독립적인 반환값 주소**를 사용하기 때문에 서로가 서로를 호출해도 문제가 발생하지 않는다.

> [!WARNING] *단, 재귀 호출은 같은 주소를 덮어쓰므로 문제가 된다.*

![[Pasted image 20260228231605.png]]

`2 + 4`의 결과를 `$00`에 저장하고, `add`의 결과인 `6`을 A 레지스터로 불러온 뒤 `sub`의 인자로 넘긴다.

`6 - 3`을 수행해 `sub`의 결과 `3`은 `$10`에 저장된다.

![[Pasted image 20260228231823.png]]

호출자가 `lda $00`, `lda $10`으로 **각 함수의 반환값을 서로 다른 주소에서 읽는다.**


## 방법 1-2: 같은 페이지 0 주소 사용

![[Pasted image 20260228231902.png]]

`ret=$00`

**모든 함수**에서 반환값을 **동일한 주소** `ret`에 저장한다.

![[Pasted image 20260228232015.png]]

동시에 `add`, `sub`의 반환값이 필요하면 레지스터 등에 **임시 저장**해서 사용하면 된다.

> [!TIP] 여러 함수에서 동시에 반환값이 필요한 경우
> 
> *`lda ret`으로 반환값을 읽은 뒤, 다음 함수를 호출하기 전에 다른 곳에 보관해야 한다.*
