# 스택 원상복구 방법

## 스택 원상복구 방법

![[Pasted image 20260228211106.png]]

`pla` 세 번 호출하는 것보다 빠른 방법이 있다.

| | 방법 1 | 방법 2 | 방법 3 |
|---|---|---|---|
| 코드 | `pla` x 3 | `tsx`/`txa`/`adc #3`/`tax`/`txs` | `tsx`/`inx` x 3/`txs` |
| **바이트** | 3 | 6 | 5 |
| **사이클** | 12 | 10 | 10 |

**방법 1**: `pla`를 인자 수만큼 호출한다.
- *인자 수가 적을 때는 간단하지만, 늘어날수록 사이클이 비례 증가한다.*

**방법 2**: `tsx`로 스택 포인터를 가져온 뒤 `adc`로 인자 수만큼 더해 복원한다.
- *`tsx` → `txa` → `adc #3` → `tax` → `txs` 순서다. 스택 포인터는 A 레지스터로 직접 가져올 수 없으므로 X 레지스터를 거쳐야 한다.*
- **인자 수가 늘어날수록 효율적이다.** *`adc`의 즉시값만 바꾸면 되므로 바이트/사이클이 일정하다.*

![[Pasted image 20260228211301.png]]

**방법 3**: `tsx`로 스택 포인터를 가져온 뒤 `inx`로 하나씩 증가시킨다.
- *인자 수만큼 `inx`를 반복해야 하므로, 방법 2에 비해 인자 수가 늘어날수록 비효율적이다.*


## 모든 인자를 스택으로 전달하면 좋은 점

![[Pasted image 20260228211515.png]]

- 함수 호출자가 **함수 내부 구현에 대해 알 필요가 없음**
- **재귀 함수를 포함한 연쇄적인 함수 호출**도 자유로움
- 각 함수 속에서 **페이지 0을 자유로이 사용**할 수 있음
  - 지역 변수 저장 용도로 활용 가능
  - *단, 다른 함수 호출 전에 페이지 0의 내용을 잘 보관해 놔야 한다*

**페이지 0**이 자유로워진다. *페이지 0으로 인자를 전달하는 방법과 비교했을 때 상대적으로 자유롭다는 뜻이다.*

페이지 0을 함수의 지역 변수를 저장하는 곳으로 사용한다면, 연쇄 호출에서 다른 함수를 호출하기 전 **지역 변수의 내용을 다른 곳에 보관**해야 한다.


## 예: 지역 변수를 스택에 보존하기

![[Pasted image 20260228211806.png]]

`func0`의 호출자(메인 함수)가 `lda #2` → `pha` → `lda #1` → `pha` → `jsr func0`으로 인자를 전달한다.

![[Pasted image 20260228211830.png]]

![[Pasted image 20260228211947.png]]

`lvar0`, `lvar1`은 **페이지 0의 지역 변수**다. (`lvar0=$10`, `lvar1=$11`)

![[Pasted image 20260228212010.png]]

연산 결과를 `sta lvar0`, `sta lvar1`로 지역 변수에 저장한다.
- *이 값은 `func1` 호출 후에도 그대로 사용해야 한다.*

![[Pasted image 20260228212056.png]]

`func1`을 호출하기 **전에** 지역 변수를 스택에 저장한다.
- `lda lvar0` → `pha` → `lda lvar1` → `pha`

![[Pasted image 20260228212130.png]]

`jsr func1`을 호출한다.
- *`func1`이 내부에서 무엇을 하는지 모른다(black box). `lvar0`, `lvar1`과 같은 페이지 0 주소를 사용할 수도 있다.*

![[Pasted image 20260228212253.png]]

`func1`에서 `rts`로 돌아온 후 **지역 변수를 복구**한다.
- `pla` → `sta lvar1` → `pla` → `sta lvar0`

> [!TIP] 고수준 언어의 지역 변수
> 고수준 언어에서는 **지역 변수를 스택에 저장**하는 방식으로 동작한다.
> *이것이 가장 범용적인 방법이기 때문이다.*


## 함수 호출

![[Pasted image 20260228212534.png]]

함수 호출은 다음 세 단계로 구성된다.

1. 현재 사용 중인 **지역 변수를 스택에 보존**
2. **데이터 전달** + `goto` (점프)
3. 회귀 후 스택에 보존된 **지역 변수를 다시 가져와** 연산


## 복습 퀴즈

### 6502 어셈블리어에서 임시 저장을 위해 스택을 사용하는 것과 제로 페이지를 사용하는 것의 주요 차이점은 무엇인가요?

> [!question]- 1. 스택이 제로 페이지보다 더 빠르다.
> 인자를 불러올 때 `lda zp` 모드로 불러오느냐 `pla`로 불러오느냐 정도 차이인데, **제로 페이지를 사용하는 것이 더 빠르다.**

> [!question]- 2. 제로 페이지는 간접 주소 지정을 허용하지만, 스택은 그렇지 않다.
> 간접 주소 지정과 관계없다.

> [!success]- 3. 스택은 재귀 함수 호출에 더 적합하다.
> **정답.** *재귀 함수를 포함한 함수의 연쇄 호출에 유리하다.*

> [!question]- 4. 제로 페이지는 반환 주소를 저장하는 데만 사용된다.
> 인자를 저장할 때도 사용된다.


### 왜 스택이 제로 페이지보다 재귀 함수 호출에 더 적합한가요?

> [!question]- 1. 스택은 제로 페이지보다 더 많은 데이터를 저장할 수 있다.
> 둘 다 크기는 동일하다.

> [!success]- 2. 스택은 중첩 호출과 반환을 자동으로 처리한다.
> **정답.** *스택이 순서를 보장해주기 때문이다.*

> [!question]- 3. 스택은 제로 페이지보다 접근 속도가 빠르다.
> 일반적으로 더 느리다.

> [!question]- 4. 스택은 제로 페이지와 다른 메모리 영역에 위치한다.
> 말 자체는 맞으나 재귀 함수 호출에 더 적합한 이유와 관계없다.


### 다음 코드를 읽고 질문에 답하세요.

```asm
caller:
	lda #$04 ; 첫번째 인자
	pha
	lda #$03 ; 두번째 인자
	pha
	jsr function
	pla ; 반환값
	sta result
	rts

function:
	pla
	sta $00 ; 돌아갈 주소 임시 저장(Low byte)
	pla
	sta $01 ; 돌아갈 주소 임시 저장(High byte)
	pla ; 두번째 인자 $03
	pla ; 첫번째 인자 $04
	sec
	sbc #$03 ; A 레지스터의 값은 $01
	pha ; 반환값에 $01이 들어간다.
	lda $01
	pha ; 돌아갈 주소 복원
	lda $00
	pha ; 돌아갈 주소 복원
	rts
```

`caller`가 완료된 후 `result`에 저장된 값은 무엇인가요?

> [!success]- 정답
> **`$01`**
> *`$04` - `$03` = `$01`. `function`이 반환값 `$01`을 스택에 push하고, `caller`가 `pla`로 꺼내 `result`에 저장한다.*
