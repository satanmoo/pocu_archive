# 워드 크기 이상의 반환값 전달

![[Pasted image 20260301002506.png]]

워드 크기 이상의 반환값을 전달하는 방법은 크게 2가지다.
1. **스택**을 사용
2. **페이지 0**에 연속해서 저장

- *레지스터는 워드 크기와 동일하기 때문에 사용할 수 없다.*


## 다시 보는 방법 2: 스택 사용 (16비트 반환값)

![[Pasted image 20260301002632.png]]

`addsub` 함수는 두 인자의 **합(+)과 차(-)** 를 동시에 계산하여 **반환값이 2개**다.

![[Pasted image 20260301002738.png]]

인자를 넘기기 전에 `pha`를 2번 호출하여 **반환값 자리 2바이트를 미리 확보**한다.
- `pha ; - 반환값 자리` → 먼저 push *(스택 더 깊은 곳)*
- `pha ; + 반환값 자리` → 나중에 push *(스택 더 얕은 곳)*

![[Pasted image 20260301002747.png]]

`lda #3` → `pha` → `lda #2` → `pha`로 인자를 넘긴다.

![[Pasted image 20260301002816.png]]

함수 내부의 스택 구조:

| 오프셋 | 내용 |
|---|---|
| `PG1+1`, `PG1+2` | 돌아갈 주소 (`jsr`이 저장) |
| `PG1+3` = `s[0]` | 첫 번째 인자 (`2`) |
| `PG1+4` = `s[1]` | 두 번째 인자 (`3`) |
| `PG1+5` = `s[2]` | **+ 반환값 자리** (합: `2 + 3 = 5`) |
| `PG1+6` = `s[3]` | **- 반환값 자리** (차: `2 - 3 = -1`) |

- `sta PG1+5,x`로 합(+) 결과를 `s[2]`에 저장한다.
- `sta PG1+6,x`로 차(-) 결과를 `s[3]`에 저장한다.

![[Pasted image 20260301003121.png]]

`pla` 2번으로 인자 2개를 제거한 뒤:
- `pla ; + 반환값`으로 **합 결과**를 먼저 꺼낸다.
- `pla ; - 반환값`으로 **차 결과**를 나중에 꺼낸다.

> [!INFO] x86 고수준 언어와 함수의 반환값
> x86 고수준 언어들도 **워드 크기를 넘는 반환값은 스택을 통해서 반환**한다.
