코드 끝(`sta out`)부터 `$FFFB` 까지 전부 `NOP`(`$EA`)으로 채운다.

```
메모리 맵: 

$8000 ~ $800B:   실제 코드
$800C ~ $FFFB:  NOP NOP NOP NOP...   ($EA)
$FFFC:   [00]    ← . WORD $8000의 LL
$FFFD:  [80]    ← . WORD $8000의 HH
$FFFE:  [00]    ← . WORD $0000의 LL
$FFFF:  [00]    ← . WORD $0000의 HH
```

 가면서 `nop`를 계속 실행하고, 주소로 `$FFFC` 에 도달한다.
- `0x00`부터 읽겠죠? 리틀 엔디언
- `0x00`을 실행하면?
- opcode `BRK`가 실행된다.
	- `BRK`의 동작은?
		- PC 레지스터의 값을 `$FFFE`, `$FFFF`로부터 읽어온 주소로 변경
		- 0x0000
		- `$0000`으로 점프한다. 예상치 못한 동작. 애초에 `$8000`부터 시작 아닌가..?

![[Pasted image 20260119021358.png]]

마찬가지로 주소로 `$FFFC` 에 도달한다.

opcode `BRK`가 실행된다.
- PC 레지스터의 값은 `$800D` 이 위치의 명령어는 `brk` 따라서 무한 루프

```
$8000:  ldx #$FF      ; 2바이트
$8002:  txs           ; 1바이트
$8003:  lda #'H       ; 2바이트
$8005:  sta $4000     ; 3바이트
$8008:  lda #'i       ; 2바이트
$800A:  sta $4000     ; 3바이트
$800D:  brk           ; 1바이트  ← 여기가 BRK! 
```
