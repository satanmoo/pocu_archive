C 코드로 `++i;`, `--i;` 같은 연산을 말한다.

> 잦은 연산이라 별도의 명령어를 제공

`adc`로 처리해도 될 것 같지만, 별도의 명령어를 제공한다.

> num = num + 1 보다 빠름
> 특히 메모리에 있는 값을 증감할 경우

### 더 느린 연산과 비교하기

![[Pasted image 20260131173549.png]]

*inc zp* 기준으로 동일한 과정을 구현하려면 아래와 같다.
- *lda zp* 로 *A 레지스터*에 값을 로드한다.
- *adc 즉시 모드* 로 1을 증가시킨다.
- *sta zp* 로 *A 레지스터*의 값(1이 증가된 값)을 메모리에 저장한다.

3 + 2 + 3 = 8 사이클

*adc 즉시 모드*로 1을 피연산자로 더하면 된다.

2 사이클이다.

`inx`, `iny` 와 사이클 수가 동일하다. 이미 `adc`로 충분히 빠르게 연산할 수 있으니 굳이 새로운 연산을 만들 필요가 없겠구나 생각하면 된다.
- 이 때문에 최적화를 위해 어셈블리어 명령어 메뉴얼을 업데이트하는 것이 좋다. 하드웨어 만드는 곳에서 니모닉을 추가하면서 최적화 하는 경향이 었기 때문이다.
- 새로 연산을 만들면 회로도 만들고 트랜지스터 체인도 추가해야하고... 비용이다.

> [!TIP] 컴파일러가 이런 최적화를 해준다. 즉 어셈블리어를 잘 조합 해준다.
