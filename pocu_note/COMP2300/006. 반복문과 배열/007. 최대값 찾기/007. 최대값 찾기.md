## 예: 최댓값 찾기

![[Pasted image 20260218222738.png]]

이 코드는 `LEN`의 값이 최소 1임을 가정한다.

### 직접 구현한 최댓값 찾기

```asm
; 길이가 1 이상인 배열에서 최대값을 구하는 코드  
    .PROCESSOR 6502  
    .ORG $8000  
  
MAX=$1000   ; 출력 변수  
LENGTH=4    ; 배열의 길이  
  
; --- 스택 초기화  
    ldx #$FF  
    txs  
; --- 배열의 요소를 순회해서 최대값을 구하는 코드  
    ldx #LENGTH-1  
    lda nums,x      ; 첫 번째 요소를 최대값으로 설정  
    dex             ; 다음 인덱스로 이동  
.loop:  
    bmi .done       ; x < 0이면 종료  
    cmp nums,x      ; 현재 최대값과 비교  
    bcs .skip       ; 현재 최대값이 크거나 같으면 스킵  
    lda nums,x      ; 새로운 최대값 로드  
.skip:  
    dex             ; x 감소  
    bpl .loop       ; x >= 0이면 반복  
.done:  
    sta MAX  
.end:  
    jmp .end  
  
;--- 컴파일 타임에 배열 선언  
    .ORG $C000,0  
nums:  
    .BYTE $00,$05,$02,$03  
  
    .ORG $FFFC,0  
    .WORD $8000  
    .WORD $0000
```

#### 어셈블 도중에 배열의 길이 구하기

![[Pasted image 20260219014311.png]]

어셈블러에서 제공하는 기능을 기반으로 배열의 길이를 구할 수 있다.
- 배열의 요소가 1바이트이기 때문에 가능한 연산이다.

![[Pasted image 20260219014411.png]]

> lda nums

*A 레지스터* 에 배열의 첫 번째 요소(`nums[0]`)를 초기 최대값으로 설정한다.
- `nums[0]`은 비교 대상이 아니라 기준값이므로, 루프는 `nums[LEN-1]`부터 `nums[1]`까지만 순회한다.

![[Pasted image 20260219014748.png]]

> ldx \#LEN-1

*X 레지스터* 의 값을 `LEN-1` 로 초기화한다. 배열을 마지막 요소부터 순회하기 위함이다.

![[Pasted image 20260219014846.png]]

> beq exit

C 코드 조건식을 그대로 어셈블리어로 변환했다.

`dex` 이후 X가 0이 되면 Z 플래그가 세워지고, `beq exit`가 이를 감지해 루프를 탈출한다.
- `lda nums`로 `nums[0]`을 이미 초기 최대값으로 설정했으므로, 색인 0은 루프에서 따로 비교할 필요가 없다.

![[Pasted image 20260219015103.png]]

> cmp nums,x
> bcs continue

색인 X의 요소와 *A 레지스터* 에 저장된 지금까지의 최대값을 비교한다.

`cmp`는 내부적으로 `A - nums[X]`를 수행하며, `A >= nums[X]`이면 C=1(Carry Set)이 된다. `bcs continue`는 이 경우 현재 최대값을 유지하고 다음 루프로 진행한다.

![[Pasted image 20260219015250.png]]

> lda nums,x

"A < nums\[X]"를 만족하는 케이스이다. *A 레지스터* 의 최대값을 갱신한다.

![[Pasted image 20260219015340.png]]

> continue:
> 	dex
> 	jmp loop

다음 루프를 진행하기 위해 색인을 감소시키고 점프한다.

![[Pasted image 20260219015429.png]]

> sta max

출력 변수에 최대값을 저장한다.
