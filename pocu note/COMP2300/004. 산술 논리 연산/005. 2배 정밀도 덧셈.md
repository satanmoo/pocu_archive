## 2배 정밀도 덧셈 예

*2배 정밀도 덧셈 *은 [[002. 최상위 비트 받아올림의 처리#최상위 비트 받아올림의 처리]] 에서 언급한 두 가지 방법 중 하나이다.
- *오버플로* 방식과 차별점은 값을 잃지 않는다는 것이다.
- *Carry Bit* 를 사용하잖아!

![[Pasted image 20260128010235.png]]

### 코드를 한 줄 씩 이해해보자.

```asm
NUM0=%01101000
NUM1=%10011010
```

[[021. 상수, 포인터 변수#상수 - 상수형 변수]]] 에서 설명한 것 처럼 *상수형 변수*를 사용할 수 있는 "주소 지정 모드"는 *즉시 모드* 뿐이다.
- 다른 "주소지정 모드"를 사용할 수 있다. 하지만 *상수형 변수*를 사용할 수 없겠죠?

```asm
out1 = $2000
outh = out1 + 1
```

[[021. 상수, 포인터 변수#변수명]] 에서 설명한 *포인터 변수*를 사용했다. 메모리의 주소에 이름을 붙이는 개념이다.

`outh = $2001` 과 동일하다.

```asm
ldx #$FF
```

`ldx`: *X 레지스터*에 값을 로드하는 명령이다. *즉시 모드*로 *X 레지스터* 에 값 `0xFF` 이 저장된다.

`0xFF` 는 특별한 값이다. [[015. 6502의 메모리 페이지 레이아웃]] 에서 설명한 *스택 메모리* 페이지에 속하는 주소 중 최대값이다.
- 스택의 시작 지점을 가리킨다.
	- [[025. S 레지스터#색인 레지스터 - S]] 에서 스택의 시작 지점이 `0xFF` 고 "push" 하면 값이 1 줄어든다고 설명했다.
