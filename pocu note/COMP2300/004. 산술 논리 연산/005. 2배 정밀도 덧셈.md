## 2배 정밀도 덧셈 예

*2배 정밀도 덧셈 *은 [[002. 최상위 비트 받아올림의 처리#최상위 비트 받아올림의 처리]] 에서 언급한 두 가지 방법 중 하나이다.
- *오버플로* 방식과 차별점은 값을 잃지 않는다는 것이다.
- *Carry Bit* 를 사용하잖아!

### 코드를 한 줄 씩 이해해보자.

```asm
NUM0=%01101000
NUM1=%10011010
```

[[021. 상수, 포인터 변수#상수 - 상수형 변수]]] 에서 설명한 것 처럼 *상수형 변수*를 사용할 수 있는 "주소 지정 모드"는 *즉시 모드* 뿐이다.
- 다른 "주소지정 모드"를 사용할 수 있다. 하지만 *상수형 변수*를 사용할 수 없겠죠?

```asm
out1 = $2000
outh = out1 + 1
```

[[021. 상수, 포인터 변수#변수명]] 에서 설명한 *포인터 변수*를 사용했다. 메모리의 주소에 이름을 붙이는 개념이다.

`outh = $2001` 과 동일하다.

```asm
ldx #$FF
```

`ldx`: *X 레지스터*에 값을 로드하는 명령이다. *즉시 모드*로 *X 레지스터* 에 값 `0xFF` 이 저장된다.

`0xFF` 는 특별한 값이다. [[015. 6502의 메모리 페이지 레이아웃]] 에서 설명한 *스택 메모리* 페이지에 속하는 주소 중 최대값이다.
- 스택의 시작 지점을 가리킨다.
	- [[025. S 레지스터#색인 레지스터 - S]] 에서 스택의 시작 지점이 `0xFF` 고 "push" 하면 값이 1 줄어든다고 설명했다.

```asm
txs
```

*Transfer X To Stack Pointer* 
- [[024. X, Y 색인 레지스터]] 참고

현재 *X 레지스터* 의 값은 `0xFF` 다. 즉 스택의 시작 주소이다. 이 값을 *스택 포인터*에 복사한다.

```asm
ldx #$FF
txs
```

이 두 줄은 스택 초기화의 정석 패턴이다.

```asm
clc
```

캐리 플래그를 0으로 초기화

다음에 `adc` 할 때 순수한 덧셈이 되도록 보장한다.

>  A + M + C -> A, C
>    
>    [ADC]

```asm
lda #NUM0
```

*A 레지스터(누산기)* 에 *즉시 모드* 로 값 `0b01101000`를 로드한다.
- 연산의 결과가 반복적으로 누적되기 때문에 누산기라고 부른다.

덧셈 연산의 피연산자를 먼저 누산기에 로드하는 개념이다.

```asm
adc #NUM1
```

피연산자 `0b10011010` 와 이미 누산기에 로드된 값 `0b01101000` 그리고 *캐리 비트* 까지 더한다.
- *캐리 비트* 는 `clc` 로 0으로 초기화 해놨지

덧셈의 결과는 `0b00000010`다. 이 값은 누산기에 저장된다.

8비트를 초과하기 때문에 *캐리 비트*의 값은 1이 된다.

```asm
sta outl
```

여기서 사용된 주소 지정 모드는 *절대 모드* 이다.

우선 `#`이 안 붙어 있기 때문에 *즉시 모드*는 아니다.
- `sta`는 *즉시 모드* 를 지원하지 않는다.

`outl` 의 값은 `$2000`다. 여기서 중요한 것은 값이 2비트 라는 것이다.

`sta` 뒤에 2비트 주소가 오면 *절대 모드*이다.
- 1비트 주소가 오면 *Zero Page*

따라서 2비트 주소 `0x2000` 에 누산기의 값(`0b00000010`)을 저장한다.

```asm
lda #0
```

누산기에 값 0을 로드한다. 누산기 값이 초기화 되었다.
- *즉시 모드*

`sta` 로 메모리에 누산기의 값을 저장했으니 클리어 하는 개념이다.

```asm
adc #0
```

*즉시 모드*

피연산자(`0b00`) 과 기존의 누산기의 값(`0b00`) 그리고 캐리 비트(`0b01`) 까지 더하고 누산기에 저장한다.
- `0b01` 이 누산기에 저장된다.

두번째 바이트의 값이다.
- 자릿수가 올라가는 개념

```asm
sta outh
```

누산기의 값(`0b01`)을 `0x2001` 주소에 저장한다.
- *절대 모드*

리틀 엔디언에 따라 높은 주소에 높은 자리수(상위 바이트)의 값이 저장되었다.
