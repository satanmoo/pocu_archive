
```asm
    .PROCESSOR 6502
    .ORG $8000

out=$4000
    ldx #$FF
    txs

    clc

    ; 1. pos + neg = pos
    lda #%00101101  ; 45
    adc #%11011111  ; -33

    sta out

    clc

    ; 2. pos + neg = neg
    lda #%00101101  ; 45
    adc #%11001110  ; -50
    sta out

    clc

    ; 3. pos + pos (no overflow)
    lda #%00101101  ; 45
    adc #%01010000  ; 80
    sta out

    clc

    ; 4. pos + pos (overflow)
    lda #%00101101  ; 45
    adc #%01010011  ; 83
    sta out

    clc

    ; 5. neg + neg (no overflow)
    lda #%10011100  ; -100
    adc #%11101100  ; -20
    sta out

    clc

    ; 6. neg + neg (overflow)
    lda #%10011100  ; -100
    adc #%11100011  ; -29
    sta out

    clc

    ; 7. result is zero
    lda #%10011100  ; -100
    adc #%01100100  ; 100
    sta out
    
    .ORG $FFFC,0
    .WORD $8000
    .WORD $0000
    
```

## 복습 퀴즈

```asm
lda #$10 
adc #$20
```

### 위 코드를 실행 후 누산기에 저장된 결과는 무엇인가요?

캐리 플래그 값을 모르기 때문에 "정의되지 않음"

### 위 코드 전에 clc 니모닉을 추가하면 누산기의 결과는 무엇인가요?

```asm
clc
lda #$10 
adc #$20
```

이렇게 되고 캐리 플래그가 0이된다.

따라서 `0x30`

### 이제 #$10, #$20을 #$80, #$90으로 바꾸면 누산기의 결과는 무엇인가요?


```asm
clc
lda #$80
adc #$90
```

비트로 바꿔보자

`0b10000000`
`0b10010000

*결과 비트* 는 `0b00010000` 다. 그리고 받아올림이 발생해 캐리 플래그가 1이 된다.

답은 `0x10`

### #$80, #$90을 더한 후 NZCV 플래그 값은 어떻게 되나요?

위에서 캐리 플래그는 구했다.

*결과 비트* 는 `0b00010000` 다.  "bit 7"이 0이라 음수 플래그는 켜지지 않는다.

결과 비트가 0이 아니기 때문에 제로 플래그는 켜지지 않는다.

결과 비트의 "bit 7"은 0이다. 

*A 레지스터* 의 "bit 7"은 1이다. 피연산자의 "bit 7"은 1이다. 두 값은 동일한데 결과 비트의 "bit 7"과 값이 다르다. 오버플로우 플래그가 켜진다.
