## sbc

![[Pasted image 20260131143654.png]]

> sbc

substract with carry
- 캐리 비트를 이용한 빼기
- 개념적으로 받아 내림

> 두 8비트 정수 간 뺄셈을 수행
> - A 레지스터
> - 피연산자 (메모리)

덧셈과 유사하다.
- "A - M" 순서에 조심하자.

> 결과: A 레지스터에 다시 대입

덧셈과 유사하다.

> V 플래그: signed 오버프롤가 발생하면 켜짐
> - 결과가 [-128, 127] 범위를 넘어서면 켜짐

[[007. 오버플로 플래그 V#signed 덧셈의 가능한 조합]] 에서 본 것처럼 "결과 비트", "A 레지스터 비트", "메모리 비트" 논리가 그대로 적용된다.

> C 플래그
> - C == 1 : 결과가 0이상. 받아내림이 없었음
> - C == 0 : 결과가 음수. 받아내림이 있었음
> - 따라서 새로운 수의 뺄셈을 하기 전에 sec를 호출해야 함

 `sbc` 이후 C 플래그 값이 어떻게 되는가에 대한 설명이다.

"결과 비트"(*A 레지스터* 에 저장되는 값)가 0 이상이라면, 개념적으로 받아내림이 없다는 말이다.
- 이 때 "캐리 플래그" 값은 1이 된다.

"결과 비트"가 음수라면("bit7"이 1), 개념적으로 받아내림이 있었다는 말이다.
- "M > A" 가 성립
- 이 때 "캐리 플래그" 값은 0이된다.

`sec` 니모닉은 "set carry flag" 캐리 플래그를 1로 설정한다.
- 뺄셈 입장에서 받아 내림이 없다는 개념

> [!NOTE] 덧셈에서는 받아 올림이 있을 때 캐리 비트가 1이 되고, 뺄셈에서는 받아 내림이 없을 때 캐리 비트가 1이 된다.

- "A > M"가 성립
- 받아 내림이 없다
- 캐리 플래그가 1이 된다.

### sec

![[Pasted image 20260131152506.png]]

>C를 1로 시작 (새로운 뺄셈 시작)

"캐리 플래그"를 1로 초기화 한다.

새로운 뺄셈을 할 때 받아 내림이 없음을 개념적으로 나타낸다.

### sbc #$02

![[Pasted image 20260131152556.png]]
> M의 부호를 뒤집음

뺄셈은 내부적으로 "부호를 뒤집고" 더하는 식으로 동작한다.

`sbc #$02` 는 명령어 1줄이지만, 내부적으로 2개의 연산을 수행한다
- 첫번째로 M의 부호를 뒤집고, "캐리 플래그"와 뒤집은 M을 더함
	- 이 연산은 2의 보수를 구하는 방법?!

개념적으로 "2를 빼겠다"를 "-2를 더하겠다"로 바꾸었다.

![[Pasted image 20260131153008.png]]

> A에 더함

`sbc`의 두번째 연산 *A 레지스터* 와 첫번째 연산의 결과를 더한다.

더해보면 "bit 7"에서 받아 올림이 발생하는 것을 알 수 있다. 따라서 "캐리 플래그"는 1이 된다.
- 이게 개념적으로 "받아 내림이 없다"를 나타낸다.

`sbc`의 2개의 연산은 CPU 1사이클에 처리된다. 하드웨어 단(ALU)에서 처리해주는 개념이다.
- 참고로 입력값이 들어오면 ALU에서 지원하는 모든 계산을 한 번에 하고, 그 중 opcode로 연산을 골라 뺄셈 결과를 얻는다.
- *멀티 플렉서* 방식
