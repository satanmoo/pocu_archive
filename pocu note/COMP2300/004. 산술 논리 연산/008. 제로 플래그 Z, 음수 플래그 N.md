## 제로 플래그 - Z (bit 1)

![[Pasted image 20260131014759.png]]

"Z"는 "Zero"의 "Z"이다

[[026. 프로세서 상태 레지스터]] 에서 위치가 "bit 1"이다.

> 8 비트가 모두 0이면 켜짐

*연산 결과* (8 비트)가 모두 0이면 켜진다.
- 연산 결과는 니모닉 별로 다르게 취급한다.

![[Pasted image 20260131015421.png]]

> 모든 데이터 이동 및 계산 연산에 적용됨

"데이터 이동"을 수행하는 니모닉으로 `lda` 같은 것들이 있다.

###  `lda`와 제로 플래그

![[Pasted image 20260131015509.png]]

메모리에서 값을 읽어와 *Accumulator*(A 레지스터)의 값을 변경한다.

여기서 *연산 결과* 는 읽어온 값이다.

즉 메모리에서 읽은 값이 0이면 "제로 플래그"가 켜진다.

메모리의 값을 읽었을 때 0이면 "제로 플레그"가 켜진다. 따라서 읽은 값을 0과 비교하는 연산 없이 조건문을 구현할 수 있다.
- "제로 플래그"가 켜졌으니 무엇을 해라
- 이 때문에 "컴퓨터에서 0과 비교하는게 빠르다" 라는 말을 할 수 있다.

![[Pasted image 20260131020506.png]]

> 결과가 0이면 켜짐

  *연산 결과* (8 bit)가 0이면 켜진다.

## "음수" 플래그 - N (bit 7)

![[Pasted image 20260131020647.png]]

> "음수"

따옴표는 음수를 의미하지만 엄밀히 말하면 음수가 아니다는 것을 강조하는 표현이다.

> bit 7이 1이면 켜짐

*연산 결과* 의 bit 7이 1이면 켜진다.

> 모든 데이터 이동 및 데이터 산술 연산에 적용

`lda` 같은 니모닉에 적용된다.
- 이동

> 주 용도는 당연히 음수 판단

"signed bit"의 음수 판단이 주 용도다.

하지만 내가 사용하는 8 bit가 "signed"라는 보장이 있나?

> 실제 음수란 뜻은 아님
> 프로그래머가 0 ~ 255 수를 더한다면 이건 양수

"unsigned"로 프로그래머가 취급하면 개념적으로 양수다.
- 하드웨어 플래그인 음수 플래그는 켜지지만, 개념적으로 양수다.

"음수 판단은 빠른 연산이다." >> 음수 플래그를 읽으면 되기 때문이다.

![[Pasted image 20260131021119.png]]

> 결과가 음수일 때 켜짐

엄밀히 말하면 음수가 아니라 bit 7이 1일 경우
