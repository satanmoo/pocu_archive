## 상세 동작표: bxx (분기 X)

![[Pasted image 20260204020101.png]]

`bxx` 명령어를 실행했는데, 조건을 만족 못해서 분기하지 않고 바로 아래 명령어를 실행하는 상황을 말한다.

> 클럭 1

`bxx` 명령어 Fetch

아직 분기 명령어인지 모른다

> 클럭 2

*Decode* 결과 분기문으로 해석했다.

동시에 Fetch 한 값은 오프셋으로 처리한다.

2 사이클 만에 모든 필요한 내용을 읽었군!

> 클럭 3

내부 연산(*Control Unit*) 에서 상태 플래그를 검사했다.

분기가 발생하지 않는다고 결론이 난다.

이와 동시에 *Fetch* 한 *opcode* 는 다음 사이클에서 실행된다.
- 한 사이클 overlapping

분기하지 않으면 2사이클 이구나!

## 상세 동작표: bxx (분기 O)

![[Pasted image 20260204020725.png]]

> [!NOTE] *사이클 2*  까지는 *분기 X* 와 동일하다.

> 클럭 3

내부 연산(*Control Unit*) 에서 상태 플래그를 검사했다.

분기가 발생한다고 결론이 난다.

> PCL + addr

동시에 오프셋을 이용해 이동할 주소를 계산한다. *PC* 가 업데이트 된다.

*PCL* 은 프로그램 카운터의 낮은 바이트(하위 8비트)를 지칭한다.
- *offset* 의 범위가 반 페이지라서 실제 덧셈은 낮은 바이트에서만 발생하죠?

이와 동시에 *Fetch* 한 *opcode2* 는 분기가 발생하지 않을 때 실행한다. 따라서 의미가 없다.
- 회색 표시

> 클럭 4

업데이트된 *PC* 에 따라 분기 후 명령어를 실행한다.

그리고 *opcode3* 을 *Fetch* 한다.

결국 회색 표시에서 사이클 낭비가 있다.

분기가 발생하면 총 3사이클이 필요하다.

> [!quote]  
> 
> 고수준 언어의 컴파일러에서 조건문의 if-else 를 뒤집어라고 힌트를 주기도 하고, 내부적으로 뒤집기도 한다. 자주 실행되는 코드를 분기하지 않고 바로 실행되게 바꾸는 것이다. 
> 
> 최신 CPU에서는 *분기 예측* 을 수행한다. 미리 분기가 어떻게 될 것인지 예상해서 명령어를 Fetch 해놓는다. 예측이 맞으면 파이프라인에 들어온 명령어를 빠르게 실행할 수 있다. 예상이 틀릴 경우 파이프라인을 초기화한다. 파이프라인 초기화의 패널티가 발생한다. 
> 
> 프로그래머는 *분기 예측* 패널티를 줄이기 위해 조건문을 직접 최적화 할 수 있다. 더 자주 일어날 것 같은 코드를 if-else 문에서 분기 없이 진행하도록 구현하는 것이다.

## 상세 동작표: bxx (분기 O, 페이지 바뀜)

![[Pasted image 20260204022403.png]]

> PCL + addr

오버플로가 발생한다. 따라서 사이클이 증가했다.

> PCH + 1 계산

오버플로가 발생해 추가적인 계산이 필요하다.

즉 페이지를 넘어간다

> [!TIP] 하위 바이트에서 오버플로가 발생하면 페이지를 넘어가게 된다. 페이지의 크기는 0xFF 이기 때문이다.

결론적으로 분기도 하고, 페이지를 넘어가면 4사이클이 필요하다.
- 클럭 5에서 `Fetch opcode3` 은 overlapping으로 날먹

> These instructions are always of 2 bytes length and perform in 2 CPU cycles, if the branch is not taken (the condition resolving to 'false'), and 3 cycles, if the branch is taken (when the condition is true). If a branch is taken and the target is on a different page, this adds another CPU cycle (4 in total).
> 
> [#### Relative Addressing (Conditional Branching)](https://www.masswerk.at/6502/6502_instruction_set.html#modes)

## 예시

![[Pasted image 20260204022839.png]]

> lda #0

*즉시 주소지정 모드* 로 *A 레지스터* 에 0을 저장한다.

> sta numpos
> sta numneg

양수의 개수, 음수의 개수를 0으로 초기화한다.

*A 레지스터*  값 0을 `$80`, `$81` 에 대입한다.

> ldx \#LEN-1

LEN 은 반복 횟수를 의미한다.

반복문을 진행하면서 *X 레지스터* 의 값은 3,2,1,0 으로 줄어든다.

> lda nums,x

*x* 는 *X 레지스터* 의 값이다.

반복문을 진행하면서 `nums[3]`, `nums[2]`, `nums[1]`, `nums[0]` 의 값을 *A 레지스터* 에 로드한다.

이 결과에 따라 *N 플래그* 가 변한다.

> bpl pos

양수면 `pos` 레이블로 분기한다.

음수면 바로 밑의 코드로 진행한다.

CPU는  *N 플래그* 를 통해 "양수/음수"를 결정한다

> inc numneg

음수일 때 `numneg` 변수의 값을 1 증가한다.
- 개수가 1개 증가

내부적으로는 메모리 `$81` 의 값을 증가시킨다.

> dex

*X 레지스터* 의 값을 1 감소시킨다.

반복문의 진행을 담당한다.

> inc numpos
> dex

`pos` 레이블은 이후에는 양수의 개수를 세는 코드다.

> [!INFO] bpl을 사용한 코드, bmi를 사용한 코드는 동일한 동작을 수행한다.
