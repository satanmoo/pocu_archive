# zp를 이용한 인자 전달

## 인자 전달 방법

![[Pasted image 20260228142707.png]]

회귀주소를 전달할 때 사용한 방법과 비슷하다.

1. 각 서브루틴에서 사용하는 레지스터들을 **Page 0에 정의**
2. **스택**을 중간(intermediate) 저장소로 사용


## 방법 1: Page 0을 사용

![[Pasted image 20260228142756.png]]

**Page 0**을 레지스터처럼 사용한다.

> [!NOTE] 네이밍 규칙
> **`p00`** — `p`는 parameter, 앞의 `0`은 0번째 함수, 뒤의 `0`은 0번째 인자
> **`v10`** — `v`는 variable(지역 변수), 앞의 `1`은 1번째 함수, 뒤의 `0`은 0번째 지역 변수

![[Pasted image 20260228143959.png]]

`mmm` 함수 내부에서 `mul`을 호출한다.
- *`mmm`의 인자 `p10`, `p11`을 `mul`의 인자 `p00`, `p01`에 복사한 뒤 `jsr mul`을 호출한다.*


### 주석

![[Pasted image 20260228144211.png]]

1. 간단한 버전

> `(p00,p01)`

인자 2개를 표현한다.

> `-> A`

반환값이 A 레지스터에 저장됨을 표현한다.

> `<X,P>`

X 레지스터와 P 레지스터(프로세서 상태 레지스터)가 변경됨을 표현한다.

2. 자세한 버전

`summary`, `arguments`, `returns`, `modifies`를 자세하게 적어준다.


### 지역 라벨

![[Pasted image 20260228144646.png]]

**범위(scope)가 있는 라벨**이다.
- 따라서 서로 범위가 다르면 지역 라벨의 이름이 **중복될 수 있다.**
- *`.loop`, `.ret` 등 `.`이 붙은 라벨이 지역 라벨이다.*

`.`이 붙지 않은 라벨은 **전역 라벨**이다.
- *어디서든 참조할 수 있다.*


### `.SUBROUTINE`

![[Pasted image 20260228145130.png]]

`.SUBROUTINE` 지시문은 **지역 라벨의 범위를 결정**한다.
- *`.SUBROUTINE` 지시문 사이가 하나의 범위가 된다.*

![[Pasted image 20260228145446.png]]

`mmm`의 `jmp .ret`에서는 자신의 `.SUBROUTINE` 범위 안에서 `.ret:`을 찾아 점프한다.
- *`mul`에도 `.ret:`이 있지만, 범위가 다르므로 영향을 받지 않는다.*


> [!WARNING] 방법 1의 한계
> **재귀 함수를 호출하면 호출자의 인자가 덮어써진다.**
> *Page 0의 고정 주소에 인자를 저장하므로, 재귀 호출 시 같은 주소를 덮어쓰게 된다. 재귀 호출이 끝나고 돌아와도 원래 인자를 복원할 수 없다.*

참고로 요즘 고수준 언어들은 제로 페이지 대신 **레지스터에 넣어서 인자를 전달**한다.
