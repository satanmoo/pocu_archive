```asm
    .PROCESSOR 6502
    .ORG $8000

num0=$4000
num1=$4001
op=$4002    ; 0 = +, 1 = -, 2 = <, 3 = >
opaddr=$10
out=$12
    ldx #$FF
    txs

    ; == DEFINE INPUTS HERE ===
    lda #12
    sta num0

    lda #16
    sta num1

    lda #1
    sta op
    ; =========================

    clc

    lda op
    asl ; x 2
    tax

    lda table,x
    sta opaddr
    lda table+1,x
    sta opaddr+1

    lda num0 ; A = solution

    jmp (opaddr)

add:
    clc
    adc num1
    jmp end

sub:
    sec
    sbc num1
    jmp end

lshift:
    ldy num1

lloop:
    beq end
    asl
    dey
    jmp lloop

rshift:
    ldy num1

rloop:
    beq end
    lsr
    dey
    jmp rloop

end:
    sta out

    .ORG $C000,0
table:
    .WORD add       ; index = 0
    .WORD sub       ; index = 2
    .WORD lshift    ; index = 4
    .WORD rshift    ; index = 6

    .ORG $FFFC,0
    .WORD $8000
    .WORD $0000
```

[[013. 코드보기 계산기]]에서 본 변수들이다.
- `num0 op num1` 식을 계산한다.


![[Pasted image 20260226003817.png]]

`op` 변수에 0~3 숫자를 저장해 연산 종류를 지정한다.

> [!NOTE] `op` 값의 의미
> - `0` → `+` (덧셈)
> - `1` → `-` (뺄셈)
> - `2` → `<` (왼쪽 시프트)
> - `3` → `>` (오른쪽 시프트)


![[Pasted image 20260226003902.png]]

**각 연산 함수의 시작 주소**를 배열(룩업 테이블)로 저장한다.
- 주소는 2바이트 크기이므로, 각 항목은 `.WORD`로 저장된다.

지정된 연산에 따라 이 테이블에서 주소를 룩업한 뒤, 그 주소로 점프할 예정이다.
- `add`로 점프하고 싶으면 `table[0]`에서 **Low 바이트**, `table[1]`에서 **High 바이트**를 읽는다.
- `sub`로 점프하고 싶으면 `table[2]`부터 2바이트를 읽는다.

> [!TIP] 점프 테이블
> **점프 테이블은 어셈블리어에서 switch/case 문을 구현하는 일반적인 기법이다.**
> *컴파일러들이 switch/case 문을 보고 점프 테이블로 최적화하기도 한다 — if문 대신 switch/case를 쓰면 얻는 장점이다.*


![[Pasted image 20260226005304.png]]

`opaddr`는 점프 테이블에서 가져온 연산 함수의 주소를 저장하는 변수다.
- **`jmp (opaddr)`** 로 **간접 절대 주소지정 모드**를 사용할 수 있다.

> [!NOTE] 간접 절대 주소지정 모드
> `jmp (opaddr)`는 `opaddr`에 저장된 값을 주소로 해석하고, 그 주소로 점프한다.
> *직접 주소를 코드에 하드코딩하지 않아도 되므로, 런타임에 분기 대상을 동적으로 결정할 수 있다.*


![[Pasted image 20260226005411.png]]

`asl`로 `op` 값을 2배 하여 `table`의 색인을 계산한다.
- *주소는 2바이트이므로, 색인은 `op × 2`가 된다. 예를 들어 `sub`이면 `op = 1`, 색인 = `2`.*

> [!NOTE] A 주소지정 모드
> `asl`에 피연산자가 없으면 **A 주소지정 모드** — *A 레지스터 자체를 피연산자로 사용한다.*


![[Pasted image 20260226010344.png]]

`tax`로 계산한 색인을 **X 레지스터**에 저장한다.
- `lda table,x`로 `table`에서 X 색인 위치의 값을 읽기 위함이다.


![[Pasted image 20260226010410.png]]

`lda table,x` / `sta opaddr`로 **주소의 Low 바이트**를 `opaddr`에 저장한다.


![[Pasted image 20260226010524.png]]

`lda table+1,x` / `sta opaddr+1`로 **주소의 High 바이트**를 `opaddr+1`에 저장한다.

> [!NOTE] 리틀 엔디언
> **6502는 리틀 엔디언** — *Low 바이트가 낮은 주소(`opaddr`), High 바이트가 높은 주소(`opaddr+1`)에 저장된다.*
> 따라서 `table+1,x`에서 High 바이트를 추가적으로 읽어야 한다.


![[Pasted image 20260226010537.png]]

점프 전에 미리 `lda num0`으로 **A 레지스터에 `num0`을 로드**한다.
- *각 연산 루틴은 A 레지스터에 첫 번째 피연산자가 담겨 있다고 가정하고 시작한다.*


![[Pasted image 20260226010604.png]]

`jmp (opaddr)`로 `opaddr`에 저장된 주소로 **간접 점프**한다.
- *이 시점에서 A = `num0`, `opaddr` = 연산 함수의 주소.*


![[Pasted image 20260226010645.png]]

`add:` 루틴부터 `rshift:` 루틴까지, **[[013. 코드보기 계산기]]의 연산 루틴과 동일하다**.
- `clc` → `adc num1` → `jmp end` 순서로 `num0 + num1`을 계산한다.
- *A 레지스터에 미리 `num0`이 로드되어 있으므로, 두 번째 피연산자 `num1`만 추가로 사용한다.*
