```asm
    .PROCESSOR 6502
    .ORG $8000

num0=$4000
num1=$4001
op=$4002    ; 0 = +, 1 = -, 2 = <, 3 = >
opaddr=$10;
out=$12
    ldx #$FF
    txs

    ; == DEFINE INPUTS HERE ===
    lda #12
    sta num0

    lda #16
    sta num1

    lda #1
    sta op
    ; =========================

    ldx op

    lda tablelo,x
    sta opaddr
    lda tablehi,x
    sta opaddr+1

    lda num0 ; A = solution

    jmp (opaddr)

add:
    clc
    adc num1
    jmp end

sub:
    sec
    sbc num1
    jmp end

lshift:
    ldy num1

lloop:
    beq end
    asl
    dey
    jmp lloop

rshift:
    ldy num1

rloop:
    beq end
    lsr
    dey
    jmp rloop

end:
    sta out

    .ORG $C000,0
tablelo:
    .BYTE <add      ; index = 0
    .BYTE <sub      ; index = 1
    .BYTE <lshift   ; index = 2
    .BYTE <rshift   ; index = 3

tablehi:
    .BYTE >add      ; index = 0
    .BYTE >sub      ; index = 1
    .BYTE >lshift   ; index = 2
    .BYTE >rshift   ; index = 3

    .ORG $FFFC,0
    .WORD $8000
    .WORD $0000
```

[[004. 코드보기 계산기(룩업 테이블)]]과 다르게 **테이블을 2개로 나눠서** 주소를 저장한다.
- *이전 코드는 한 테이블 안에 2바이트짜리 주소를 `.WORD`로 저장했지만, 이번에는 Low 바이트와 High 바이트를 각각 별도의 테이블에 `.BYTE`로 저장한다.*

`tablelo`는 **하위 바이트(Low byte)** 를 저장한다.


![[Pasted image 20260226012215.png]]

`tablehi`는 **상위 바이트(High byte)** 를 저장한다.

> [!TIP] 테이블 분리의 장점 — 256개 주소 저장 가능
> **테이블을 2개로 분리하면 최대 256개의 주소를 저장할 수 있다.**
> - *X 레지스터는 8비트이므로 색인 범위가 0~255(총 256개)다.*
> - *이전 방식은 `.WORD`(2바이트) 단위로 저장해 색인을 `0, 2, 4, ...`로 사용했으므로, 실질적으로 128개만 저장 가능했다.*
> - 테이블을 분리하면 색인을 `0, 1, 2, ...`로 사용할 수 있어 **X 레지스터의 256개 색인을 모두 활용**할 수 있다.


![[Pasted image 20260226012601.png]]

`ldx op`로 `op` 값을 **그대로 색인으로 사용**할 수 있다.
- *이전 방식에서는 `asl`로 2를 곱해야 했지만, 테이블을 분리하면 그럴 필요가 없다.*
- **`asl` 연산이 빠지므로 몇 사이클 절약**된다.


![[Pasted image 20260226012936.png]]

`lda tablelo,x` / `sta opaddr`로 **Low 바이트**를, `lda tablehi,x` / `sta opaddr+1`로 **High 바이트**를 읽어 `opaddr`에 저장한다.
- *X 레지스터 값을 `tablelo`, `tablehi`에 각각 **색인 주소지정 모드**로 사용한다.*


![[Pasted image 20260226013011.png]]

`lda num0` → `jmp (opaddr)` 이후의 **연산 루틴 코드는 [[004. 코드보기 계산기(룩업 테이블)]]과 동일하다.**
- *테이블 구조만 바뀌었을 뿐, 실제 연산 로직은 그대로다. 그러면서도 전체적으로 2사이클 정도 빨라진다.*
