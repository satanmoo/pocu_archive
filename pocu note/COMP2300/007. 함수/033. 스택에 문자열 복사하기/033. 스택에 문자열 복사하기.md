# 스택에 문자열 복사하기

![[Pasted image 20260301230017.png]]

C에서는 문자열 리터럴을 char 배열에 대입하면 컴파일러가 스택에 복사하도록 해준다. 따라서 Read-Only 메모리를 건들지 않는다.

이 때 내부적으로 memcpy를 호출한다.

## 스택에 문자열 복사하기 - 시도 1

![[Pasted image 20260301230230.png]]

> [!important] 스택에 문자열 복사 — push 루프 동작
>
> ```asm
> ldx #$FF
> pushloop:
>     inx             ; X = 0, 1, 2, 3...
>     lda str,x       ; A = str[X], Z 플래그 갱신
>     pha             ; 스택에 push (플래그 변경 없음)
>     bne pushloop    ; Z=0이면 계속, Z=1이면 탈출
> ```
>
> | 반복 | X | lda str,x | A | Z 플래그 | pha | bne |
> |------|---|-----------|---|---------|-----|-----|
> | 1 | 0 | str[0] | `'a'` | 0 | push 'a' | 분기 ✅ |
> | 2 | 1 | str[1] | `'b'` | 0 | push 'b' | 분기 ✅ |
> | 3 | 2 | str[2] | `'c'` | 0 | push 'c' | 분기 ✅ |
> | 4 | 3 | str[3] | `$00` | **1** | push $00 | 탈출 ❌ |
>
> `bne`의 Z 플래그는 **`lda`가 설정한 것**이다.
> `pha`는 플래그를 변경하지 않으므로 `lda`의 결과가 그대로 유지된다.
> 널 문자도 **push한 뒤에** 탈출하므로 스택에 널 종료 문자가 포함된다.

> [!note] 스택 포인터 저장 — tsx / stx vstr
> ```asm
> tsx            ; X = 현재 스택 포인터 (S 레지스터 값)
> stx vstr       ; 제로페이지 $00에 저장
> ```
>
> 스택의 주소는 항상 `$0100 + S`이므로,
> vstr에 S 값을 저장해두면 나중에 스택에 복사한 문자열의
> **시작 위치를 복원**할 수 있다.

![[Pasted image 20260301230511.png]]

> [!important] PG1+1을 사용하는 이유 — 스택 포인터의 특성
>
> ```asm
> ldx vstr          ; X = 저장해둔 스택 포인터(S)
> lda PG1+1,x       ; A = $0101 + X
> ```
>
> 스택 포인터(S)는 **마지막으로 push한 값이 아니라 그 아래 빈 공간**을 가리킨다.
>
> ```
> 주소      스택
> $01FC  │  'a'  │  ← push한 데이터 시작
> $01FB  │  'b'  │
> $01FA  │  'c'  │
> $01F9  │  $00  │  ← 마지막 push한 값
> $01F8  │  ??   │  ← S가 가리키는 위치 (비어있음)
> ```
>
> `PG1` = `$0100`이므로 `PG1 + S`는 빈 공간을 가리킨다.
> **+1을 해야 실제 데이터가 있는 첫 번째 위치**에 접근할 수 있다.
>
> `lda PG1+1,x` = `lda $0101,x` → `$0101 + S` = 마지막 push한 값의 위치

![[Pasted image 20260301230641.png]]

문자열을 널문자 부터 거꾸로 읽게된다.

## 스택에 문자열 복사하기 - 시도 2

![[Pasted image 20260301230707.png]]

> [!important] 역순 push — 널 문자부터 넣어서 꺼낼 때 정순으로 읽기
>
> ```asm
> ldx #estr-str-1    ; X = 3 (널 문자 위치, 마지막 인덱스)
> pushloop:
>     lda str,x       ; 뒤에서부터 읽기
>     pha              ; 스택에 push
>     dex              ; X--
>     bpl pushloop     ; X >= 0이면 계속
> ```
>
> | 반복 | X | str[X] | push 값 |
> |------|---|--------|--------|
> | 1 | 3 | `$00` | '\0' |
> | 2 | 2 | `'c'` | 'c' |
> | 3 | 1 | `'b'` | 'b' |
> | 4 | 0 | `'a'` | 'a' |
> | 끝 | -1 ($FF) | — | bpl 탈출 (N=1) |
>
> **스택 (LIFO) 결과**
> ```
> $01FF │ '\0' │  ← 가장 먼저 push (바닥)
> $01FE │  'c' │
> $01FD │  'b' │
> $01FC │  'a' │  ← 가장 나중에 push (꼭대기)
>   S → $01FB │  ??  │
> ```
>
> 스택은 LIFO이므로 나중에 `PG1+1,x`로 **위에서부터 읽으면**
> `'a'` → `'b'` → `'c'` → `'\0'` 순서로 정상적으로 읽힌다.

> [!tip] estr-str-1의 의미
> ```
> str:  .BYTE "abc"
>       .BYTE 0
> estr:
> ```
> `estr - str` = 4 (문자열 전체 길이, 널 포함)
> `estr - str - 1` = 3 (마지막 인덱스, 즉 널 문자의 위치)
>
> 어셈블러가 레이블 간 거리를 **컴파일 타임에 계산**해주므로
> 문자열 길이를 수동으로 세지 않아도 된다.

![[Pasted image 20260301230918.png]]

이제 문자열의 첫번째 문자부터 접근할 수 있다.

![[Pasted image 20260301230938.png]]

> [!important] 스택의 문자열을 대문자로 변환하는 루프
>
> ```asm
> ldx vstr             ; X = 저장해둔 스택 포인터
> upcloop:
>     lda PG1+1,x      ; 스택에서 문자 읽기, Z 플래그 갱신
>     beq exit          ; $00(널 문자)이면 탈출
>
>     and #%11011111    ; 소문자 → 대문자 변환
>     sta PG1+1,x       ; 변환된 값을 스택에 다시 저장
>     inx                ; 다음 문자로
>     jmp upcloop
> exit:
> ```

> [!note] beq exit — 널 종료 문자 감지
> `lda`가 `$00`을 읽으면 **Z=1**이 설정된다.
> `beq` = **B**ranch if **Eq**ual (Z=1일 때 분기)
> 즉 널 문자를 만나면 변환을 멈추고 exit로 빠진다.

> [!tip] and #%11011111 — 소문자를 대문자로 바꾸는 비트 트릭
>
> | 문자 | ASCII (2진수) | AND 결과 | 변환 |
> |------|--------------|---------|------|
> | 'a' | 0**1**10 0001 | 0**0**10 0001 | 'A' |
> | 'b' | 0**1**10 0010 | 0**0**10 0010 | 'B' |
> | 'c' | 0**1**10 0011 | 0**0**10 0011 | 'C' |
>
> ASCII에서 소문자와 대문자의 차이는 **비트 5 하나**뿐이다.
> `AND #%11011111`로 비트 5를 **0으로 클리어**하면
> 소문자에서 32를 빼는 것(`'a'(97) - 32 = 'A'(65)`)과 동일한 효과다.
> 뺄셈 대신 비트 연산이므로 **캐리 플래그에 영향을 주지 않는다.**

![[Pasted image 20260301231226.png]]

> [!important] 스택 복구 — 문자열이 차지한 공간을 되돌린다
>
> ```asm
> lda vstr           ; A = 저장해둔 스택 포인터 ($FB)
> clc
> adc #estr-str      ; A = $FB + 4 (널 포함 문자열 길이)
> tax                ; X = $FF
> txs                ; S = $FF (스택 포인터 복원)
> ```
>
> | 단계 | 값 | 설명 |
> |------|------|------|
> | vstr | $FB | 문자열 push 후 저장한 S |
> | estr-str | 4 | 널 문자 포함 문자열 길이 ("abc\0") |
> | $FB + 4 | **$FF** | 문자열 push 이전의 S |
>
> ```
> 복구 전                 복구 후
> $01FC │ 'A' │          $01FC │ 'A' │  ← 아직 남아있지만
> $01FD │ 'B' │          $01FD │ 'B' │    S 위에 있으므로
> $01FE │ 'C' │          $01FE │ 'C' │    "없는 것"으로 취급
> $01FF │ '\0'│          $01FF │ '\0'│
>   S → $01FB              S → $01FF ← 복원됨
> ```
>
> 스택의 데이터를 실제로 지우는 것이 아니라
> **S를 원래 위치로 되돌리면** 그 공간은 다음 push에 의해 덮어쓰여진다.
> 6502에서 스택 "정리"란 곧 **스택 포인터를 이동시키는 것**이다.
