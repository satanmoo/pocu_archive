# 코드보기: 두 배열 더하기

## 복습 퀴즈

### 6502 어셈블리어에서 절대 주소 지정이 불가능한 상황은 언제인가요?

> [!question] Q1. 6502 어셈블리어에서 절대 주소 지정이 불가능한 상황은 언제인가요?
>
> 1. 주소가 16비트 범위를 초과할 때
> 2. 주소가 스택 메모리에 있을 때
> 3. 주소가 제로 페이지 내에 있을 때
> 4. **주소가 실행 도중에 결정될 때** ✅
>
> > [!success]- 정답: 4번
> > 절대 주소 지정 모드는 명령어에 주소가 **하드코딩**되므로
> > 실행 도중 동적으로 결정되는 주소에는 사용할 수 없다.
>
> > [!failure]- 오답 해설
> > **1번**: 주소 핀이 16비트로 고정이라 16비트 범위를 초과하는 상황 자체가 불가능하다.
> >
> > **2번**: 스택은 1페이지($0100~$01FF)에 존재하므로 절대 주소 지정이 가능하다.
> >
> > **3번**: 제로 페이지는 0페이지($0000~$00FF)에 존재하므로 마찬가지로 가능하다.

### 6502 어셈블리어에서 간접 주소 지정에 제로 페이지를 사용하는 이유가 무엇인가요?

> [!question] Q2. 6502 어셈블리어에서 간접 주소 지정에 제로 페이지를 사용하는 이유가 무엇인가요?
>
> 1. 제로 페이지 주소 지정이 더 느리기 때문에
> 2. **제로 페이지 주소 지정이 더 적은 사이클을 필요로 하기 때문에** ✅
> 3. 제로 페이지가 간접 주소 지정을 지원하는 유일한 메모리 영역이기 때문에
> 4. 제로 페이지 주소 지정이 더 안전하기 때문에
>
> > [!success]- 정답: 2번
> > 제로 페이지는 피연산자가 1바이트로 충분하므로
> > 명령어 크기와 사이클이 줄어든다.
> > 간접 참조는 이미 느린 연산이라 **사이클 절약이 핵심**이다.
>
> > [!failure]- 오답 해설
> > **1번**: 오히려 빠르다. 피연산자 1바이트 → 더 적은 Fetch.
> >
> > **3번**: `jmp`는 절대 간접 모드 `(a)`도 지원한다. 제로 페이지가 유일한 영역은 아니다.
> >
> > **4번**: 안전성과는 무관하다.
