# 스택을 통한 반환값 전달

## 방법 2: 스택 사용

![[Pasted image 20260228232530.png]]

> [!INFO] **가장 범용적인 방법**이다.

![[Pasted image 20260228232652.png]]

`pha ; 반환값 자리`로 **반환값이 저장될 자리를 미리 확보**한다.
- *A 레지스터에 어떤 값이 있든 상관없다. 스택에 반환값 자리를 확보하기만 하면 된다.*
- *규약대로라면 함수(callee)가 이 위치에 반환값을 넣어 줄 것이기 때문이다.*

![[Pasted image 20260228232752.png]]

`PG1+1`, `PG1+2`에는 `jsr`이 저장한 **돌아갈 주소**가 있다.

`s[2]`는 반환값의 위치로, 돌아갈 주소와 인자를 제외한 색인이다.
- *`PG1+3` = `s[0]`, `PG1+4` = `s[1]`, `PG1+5` = `s[2]`(반환값 자리)*
- 함수에서 `sta PG1+5,x`로 **호출자가 미리 확보해 둔 반환값 자리에 결과를 저장**한다.

![[Pasted image 20260228232953.png]]

호출자로 돌아온 뒤 `pla`를 두 번 호출하면 나중에 넣은 `s[0]`, `s[1]` **인자 2개를 제거**한다.

![[Pasted image 20260228233126.png]]

`pla ; 반환값`으로 **반환값을 A 레지스터에 꺼낸다.**

![[Pasted image 20260228233156.png]]

`tay`로 `add`의 반환값을 **Y 레지스터에 임시 저장**한다.
- *다음 함수(`sub`)에서 A 레지스터를 사용해야 하기 때문이다.*

`sub` 함수의 동작은 `s[0] - s[1]`이다.
- *나중에 넣은 인자(Y 레지스터에 저장한 `add`의 반환값)에서 먼저 넣은 인자(`3`)를 뺀다.*
