# x86 Debugging Tip (.COM + DEBUG)

이 문서는 `main.asm` 기준으로, DOSBox-X + MASM 환경에서 `.DATA` 메모리 상태를 확인하는 빠른 절차를 정리한 가이드다.

## 대상 코드 요약

```asm
TITLE Main

  

.DOSSEG

.8086

.NO87

.MODEL TINY

  

.DATA

nums DB 1,2,3,4,5

mk1 DB 0A1h

array DB 20 DUP (2)

mk2 DB 0A2h

masks DB 10 DUP (30h,20h,01h,09h)

mk3 DB 0A3h

str0 DB 32 DUP ("Hello, x86!")

mk4 DB 0A4h

ddd DD 3 DUP (3 DUP (3 DUP (0)))

mk5 DB 0A5h

buffer DB 128 DUP (?)

  

.CODE

.STARTUP

; 디버거에서 여기서 멈추고 싶으면 아래 줄을 활성화

int 3

  

; 프로그램 종료

mov ah, 4Ch

xor al, al ; 리턴값 0

int 21h

END
```

`main.asm`에는 아래 데이터가 선언되어 있다.

- `nums DB 1,2,3,4,5`
- `mk1 DB 0A1h`
- `array DB 20 DUP (2)`
- `mk2 DB 0A2h`
- `masks DB 10 DUP (30h,20h,01h,09h)`
- `mk3 DB 0A3h`
- `str0 DB 32 DUP ("Hello, x86!")`
- `mk4 DB 0A4h`
- `ddd DD 3 DUP (3 DUP (3 DUP (0)))`
- `mk5 DB 0A5h`
- `buffer DB 128 DUP (?)`

코드 시작부에 `int 3`가 있어 디버거에서 즉시 멈출 수 있다.

## 빌드 및 실행

1. 어셈블

```dos
ML /W3 /Zi MAIN.ASM
```

2. 디버거 실행

```dos
DEBUG MAIN.COM
```

## DEBUG 기본 확인 순서

1. `?`
- 도움말 화면 출력
- `R`, `D`, `U`, `S`, `G` 등 사용 가능 명령 확인

2. `r`
- 레지스터 값 확인
- 현재 코드 위치는 `CS:IP`
	- 참고로 TINY 모델을 사용하기 때문에 DS, CS값이 동일하다.
![[Pasted image 20260308173311.png]]

3. `u cs:ip`
- 현재 위치부터 디스어셈블ㅕ

4. `g`
- 실행 진행(현재 코드의 `int 3`에서 브레이크)

참고: `-u`, `-r`처럼 보이는 `-`는 DEBUG 프롬프트이고, 실제 명령은 `u`, `r`이다.

## .DATA 메모리 덤프 보는 방법

1. 덤프 기본

```dos
d ds:0100
```

엔터를 누르면 다음 범위를 연속해서 볼 수 있다.

2. 마커 바이트 검색

```dos
s ds:0100 ffff a1
s ds:0100 ffff a2
s ds:0100 ffff a3
s ds:0100 ffff a4
s ds:0100 ffff a5
```

3. 찾은 주소 주변 확대

```dos
d ds:<found_addr>-20
```

`A1/A2/A3/A4/A5`를 경계로 각 데이터 블록 위치를 빠르게 구분할 수 있다.

## S(search) 명령어 보충 설명

`S`는 메모리에서 특정 바이트 패턴을 찾는 명령이다.

```dos
s <start> <end> <byte1> [byte2 ...]
```

예시:

```dos
s ds:0100 ffff a1
```

의미:

- 검색 범위: `DS:0100` ~ `DS:FFFF`
	- 시작 주소에 `ds:`를 쓰면 끝 주소는 같은 세그먼트로 해석된다.
- 검색 대상: 값 `A1h` 바이트

### 마커 바이트 검색 논리

1. 코드에 `mk1 DB 0A1h`, `mk2 DB 0A2h`처럼 유니크 값을 넣는다.
2. DEBUG는 변수 이름을 직접 모르므로, `S`로 유니크 값 주소를 찾는다.
3. 찾은 주소 주변을 `D`로 덤프하면 데이터 블록 경계를 확인할 수 있다.

실전 예:

```dos
s ds:0100 ffff a1
d ds:<found_addr>-20
```

패턴 검색을 길게 주면 오탐을 줄일 수 있다.

```dos
s ds:0100 ffff 30 20 01 09
```

위 명령은 `masks DB 10 DUP (30h,20h,01h,09h)` 패턴 찾기에 유용하다.

## D(dump) 명령어 보충 설명

`D`는 메모리를 16진수 + ASCII 형태로 보여주는 덤프 명령이다.

```dos
d [address]
d <start> <end>
```

예시:

```dos
d ds:0100
d ds:0100 ds:013f
```

### `d ds:<found_addr>-20`의 `-20` 의미

- `-20`은 찾은 주소에서 `20h` 바이트만큼 뒤로 이동하라는 뜻이다.
- DEBUG 숫자는 기본적으로 16진수이므로 `20h = 32`(10진수) 바이트다.
- 즉 마커 바로 앞 데이터까지 함께 보려고 시작 주소를 조금 당기는 표현이다.

예를 들어 `found_addr = 012A`라면:

```dos
d ds:012a-20
```

는 실제로 `DS:010A`부터 덤프를 시작하는 효과가 있다.
