## ORG

`[label] ORG	    exp[,DefaultFillVal]`

This pseudop sets the current origin.  You can also set the
global default fill character (a byte value) with this
pseudoop.  NOTE that no filler is generated until the first
data-generating opcode/psueoop is encountered after this one.
Sequences like:

org  0,255
org  100,0
org  200
dc   23

will result in 200 zero's and a 23. This allows you to specify
some ORG, then change your mind and specify some other (lower
address) ORG without causing an error (assuming nothing is
generated in between).

Normally, DS and ALIGN are used to generate specific filler
values.

> "This pseudop sets the current origin."  

지금부터 생성되는 코드/데이터가 메모리의 어느 주소에 위치할 것인가를 지정한다.

"pseudop"는 "의사 명령어"다 실제 CPU가 실행하는 명령어가 아니라, 어셈블러에게 주는 지시사항이다.

```asm
org  $1000      ; "지금부터 주소 $1000번지부터 시작해"
lda  #$42       ; 이 명령어는 $1000에 위치하게 됨
sta  $2000      ; 이 명령어는 $1002에 위치하게 됨 (lda가 2바이트라면)
```

> you can also set the global default fill character (a byte value) with this pseudoop.

-   "Fill character"은 채움 문자다. 메모리의 빈 공간을 무엇으로 채울지 지정하는 값이다.
- "Gloabl default" 의 의미
	- **전역(global)**: 한 번 설정하면 다음에 바꾸기 전까지 계속 적용됨
	- **기본값(default)**: 별도 지정 없으면 이 값을 사용

왜 채움 문자가 필요할까?

```asm
org  $1000
dc. b $AA          ; $1000에 $AA 저장

org  $1010        ; 갑자기 $1010으로 점프! 
dc.b $BB          ; $1010에 $BB 저장
```

`$1001`~`$100F` (15바이트)는 비어있다. 이 공간을 무엇으로 채울지 결정하는게 fill character이다.

> NOTE that no filler is generated until the first data-generating opcode/pseudoop is encountered after this one.

- 실제로 채움(Filling)이 발생하는 시점은 데이터를 생성하는 명령어를 만났을 때다.

주어진 예시 코드를 확인해보면 "will result in 200 zero's and a 23." (200개의 0과 하나의 23이 생성됨)

첫번째 줄의 `org  0,255` 에서 origin은 0, 채움문자는 255로 설정된다. 데이터 생성은 아직이다.

두번째 줄의 `org 100,0`에서 origin은 100으로, 채움문자는 0으로 설정된다. 데이터 생성은 아직이다.

세번째 줄의 `org 200`에서 origin은 200으로, 채움문자는 0으로 설정되어 있다. 데이터 생성은 아직이다.

네번째 줄의 `dc   23`에서 드디어 데이터 생성 명령어를 만났다. 현재 origin은 200이기 때문에 주소 0~199는 채움 문자 0으로 채우고, 200에 23을 저장한다.

> This allows you to specify some ORG, then change your mind and specify some other (lower address) ORG without causing an error

문서의 결론이다 "지연 채움"(Lazy fill) 때문에 작성자가 마음이 바뀌어 다른 주소를 지정해도 문제가 없도록 했다.


## WORD

`[label] DC[.BWL]    exp,exp,exp ...`

Declare data in the current segment.  No output is generated if
within a .U segment.  Note that the byte ordering for the
selected processor is used for each entry.

The default size extension is a byte.

BYTE, WORD and LONG are synonyms for DC.B, DC.W and DC.L.

> Declare data in the current segment.

현재 세그먼트(코드/데이터를 논리적으로 구분하는 영역)에 데이터를 선언한다.
- 메모리에 특정 값을 직접 넣는 개념

> No output is generated if within a .U segment.

".U segment"는 초기화되지 않은(Uninitialized) 세그먼트다. 자세한 내용은 생략하고 초기화되지 않은 세그먼트에는 출력할 수 없다.

> Note that the byte ordering for the selected processor is used for each entry.

선택된 프로세서의 바이트 순서가 적용된다.
- 리틀 엔디언
- 빅 엔디언

> The default size extension is a byte.

"size extension"은 크기 확장자이다. `DC`뒤에 붙는 `.B`, `.W`, `.L`을 말한다. 확장자를 생략하면 `.B`로 취급된다.

> BYTE, WORD and LONG are synonyms for DC.B, DC.W and DC.L.
