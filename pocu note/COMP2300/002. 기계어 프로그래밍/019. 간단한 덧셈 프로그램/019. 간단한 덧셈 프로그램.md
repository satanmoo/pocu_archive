## D) 최종 값을 읽어오는 코드를 추가

**Step5(ALU를 사용하는 프로그램 작성) - D**

![[Pasted image 20251230012004.png]]

`0x2200`주소에서 읽어서 **A 레지스터**에 저장하는 기계어 코드를 작성해보자.

![[Pasted image 20251230012058.png]]

`LDA a`를 사용해서, 주소 지정을 해주면 된다.
- 주소에 `0x2200`을 넣어주면 되겠지?
- 이 명령어는 `0xAD`

![[Pasted image 20251230012151.png]]

그리고 피연산자로 `0x2200`을 적어주면 된다.
- 주소 핀은 16비트니까 2바이트로 읽고 **리틀 엔디언**을 적용하면 된다.

이렇게 수정하고 다시 실행하면 **bin 파일**이 변하고, 다시 플래시하고 진행하면 됩니다.

## E) 결과 확인

**Step5(ALU를 사용하는 프로그램 작성) - E**

![[Pasted image 20251230012422.png]]

`0x8008`에서 `0xAD`를 읽었다.
`LDA a`에 따라 16비트 주소를 **리틀 엔디언**으로 읽으면, `0x2200`
`0x2200`주소에서 값을 읽으면 `0x22`
- 어 올바른 값이 아닌데?

`0x2200`이라는 주소는 ROM과 어떻게 매핑되어 있지? 메모리 매핑을 확인해보자

![[Pasted image 20251230012842.png]]

`0x2200`이라는 주소는 ROM과 매핑되지 않는다.
- [[014. ROM 주소 매핑] 참고

ROM은 **Write Enabled**상태가 아니다.
- [[016. OE, CE, WE#데이터 시트를 확인]] 참고

어딘가에 Write했는데, 저장한 값이 없는 것이다. 따라서 `0x2200`에서 읽은 값은 **가비지 값**이다.

여기서 결론은 ROM에 Write 하는 코드를 작성하면 안 된다는 것이다. 그렇다면 어떤 메모리에 값을 저장해야 하지? >> RAM
