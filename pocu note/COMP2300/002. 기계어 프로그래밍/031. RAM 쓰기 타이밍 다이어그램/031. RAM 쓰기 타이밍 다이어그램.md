## E) Write 문제가 없는지 확인하기

### tMDS

![[Pasted image 20260104014229.png]]

Write Data Delay Time: 최대 "25 ns" 딜레이 이후 CPU가 값을 쓰기 시작한다.
- RAM은 최소 "tMDS"이후 데이터를 읽을 준비가 되어야 한다.
- "525 ns"

### tDHW

![[Pasted image 20260104014417.png]]

Wirte Data Hold Time: "1000 ns"(사이클이 끝나고) 최소 "10 ns"는 데이터가 홀딩된다. 즉 쓰기가 가능하다.
- 따라서 "1010 ns"까지 데이터를 쓴다.
- 신기하게도 유효한 주소가 유지되는 기간의 끝과 동일하다.

## 만약 클럭이랑 동기화 하지 않았다면 CPU에서 RAM에 Write 할 때 어떤 문제가 생길까?

![[Pasted image 20260104023442.png]]

"Address Hold from Write End"가 최소 "0 ns"다. 주소를 바꿈과 동시에 혹은 그 전에 "CEB"를 바꿔야 한다.
- 왜???

클럭과 동기화 하지 않으면 주소와 "CEB"가 동시에 바뀐다.
- 왜???

동시에 바뀌면 약간의 딜레이가 있을 수 있다. 따라서 주소가 먼저 바뀌는지, "CEB"가 먼저 바뀌는지 알 수 없다. 클럭과 연결하면 [[031. RAM 쓰기 타이밍 다이어그램#CPU에서 "979 ns" ~ "1009 ns" 동안 올바른 데이터를 출력하냐?]] 에서 본것 처럼 "Address Hold" 즉 주소가 유효한 타이밍 보다 "10 ns"먼저 "CEB"
가 올라가는 것이 보장된다.
