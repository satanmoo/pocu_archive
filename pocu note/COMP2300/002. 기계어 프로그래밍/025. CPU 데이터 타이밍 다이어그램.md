## C) 타이밍 다이어그램 분석하기

### tDHR, tDSR

![[Pasted image 20260101202452.png]]

"1000 ns" 기준으로 "tACC"의 끝과 사이를 "tDSR"이라고 부른다.

"tDSR": "Setup" 실제 유효한 데이터가 들어오기 시작하는 타이밍

"tDHR": "Hold" 유효한 데이터가 들어왔을 때 언제까지 이 데이터를 읽을 수 있나? 보장 하는가?

RAM이 초록색 하이라이트 시간 이상으로 데이터를 주어야지 CPU는 올바르게 읽을 수 있다.

![[Pasted image 20260101203235.png]]

### 6502의 설계 철학

한 사이클을 절반으로 나눠서, 절반 사이클 전반부에 유효한 주소를 출력하고 후반부에 데이터를 읽는다.

주소 설정은 사이클의 전반부, 데이터를 읽는 것은 사이클의 후반부

중요한 것은 한 사이클 안에서 주소 설정, 데이터 읽기를 수행한다는 것이다.


### 시간 표시하기

![[Pasted image 20260101203648.png]]

표를 기반으로 타이밍 다이어그램에 구체적인 시간을 표시해보자

우리는 "5V"를 사용하기에 가장 왼쪽 열을 참고하면 된다.
- 참고로 표에서 오른쪽으로 갈 수록 주파수가 감소하는데, 전력이 줄어들어서 감소한다고 생각하면 된다.
- 오버 클럭을 사용하면 전기가 많이 들고, 클럭 수가 증가한다.


![[Pasted image 20260101203913.png]]

표에서 찾아서 최대값 기준으로 시간을 적으면 된다.

"tAH"는 "10 ns"걸린다고 표시하면 된다.
- 최소 "10 ns"정도 CPU는 주소를 유지한다.

![[Pasted image 20260101204235.png]]

"tDHR"에 따르면 최소 "1010 ns"까지 RAM아 데이터를 홀드해야 CPU가 읽을 수 있다.

"tDSR"에 따르면 "990 ns"부터 CPU는 데이터를 읽기 시작한다. 따라서 RAM은 "990 ns" 부터 유효한 데이터를 주어야 한다.

![[Pasted image 20260101204742.png]]

"20 ns"이상 RAM이 유효한 데이터를 출력하고 있어야 한다.

"30 ns" ~ "990 ns" 사이 "960 ns"나 되는 충분한 시간이 있기에, RAM이 데이터를 찾을 시간은 충분합니다.
