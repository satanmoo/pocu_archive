우리에게 중요한걸 먼저 보자.

"HIGH IMPEDANCE"
- 데이터 아웃 버스에 데이터 출력을 하지 않고 있다. 
- 가비지 값이다.

![[Pasted image 20260102015435.png]]

"OEB"가 VL로 떨어지고 "tLZOE" 시간이 지나면 데이터를 출력하지만 유효하지 않은 데이터다.
- "HIGH IMPEDANCE"와 다르게 쓰기는 쓴다.

![[Pasted image 20260102015448.png]]

"tDOE" 이후 유효한 데이터가 출력된다.

사실 이건 우리에게 중요하지 않다. 우리는 "OEB"가 항상 VL이 되도록 설계 했으니까
- [[020. RAM 설치하기#RAM OEB 연결하기]] 참고

![[Pasted image 20260102015834.png]]

유효한 데이터가 출력되는 것은 "tHZOE" 구간까지 유지된다. 또는 "tHZCE"구간까지 유지된다.
- "tHZCE"에서 "CE"는 지금 흐릿하게 보이는 "CEB"가 올라간 뒤 구간이다.
- "OEB", "CEB" 둘 중 뭐가 먼저 올라가냐에 따라 유효한 데이터가 출력되는 구간이 결정된다.
- 하지만 "OEB"값은 VL로 고정이니 우리에게 중요한 것은 "CEB"
	- 이제 스크린샷의 "CEB"가 진하게 표시됬다.

결론적으로 "CEB"가 내려갈 때 "tACE"만큼 지난 후 유효한 데이터가 나온다. 그리고 "CE"가 올라가고 "tHZCE"만큼 지날 때 까지는 유효한 데이터가 출력된다.

![[Pasted image 20260102020011.png]]

결론적으로 "CEB"와 "DATA OUT"만 보면 된다.

"CEB"가 High에서 Low가 되면서 유효한 주소가 RAM에 들어온다.
- 이것이 유효한 주소인가??
	- "CEB"에 연결된 주소 핀이 "A15"였다. CPU에서 RAM을 사용할 때 "A15"는 VL이 들어오고 따라서 "CEB"도 VL이 들어온다.
		- [[021. RAM 주소 매핑]]
	- RAM을 사용한다. ==  CPU에서 유효한 주소를 사용한다. == "CEB"가 내려온다.
		- 이렇게 동치가 된다.

"tRC": Read Cycle, 이 term의 값은 스펙을 참고하자.

![[Pasted image 20260102021602.png]]

[[025. CPU 데이터 타이밍 다이어그램]]을 참고해서 유효한 주소의 시작과 끝을 알 수 있다.
- 시작: "30 ns"
- 끝: "1010 ns"

RAM 스팩을 보면, "tRC"가 "70 ns"동안 지속되는데, CPU가 유효한 주소를 가지는 시간("980 ns")동안 충분히 지속된다.

![[Pasted image 20260102021944.png]]

"tACE"가 지나고 유효한 데이터가 RAM에서 출력된다.

![[Pasted image 20260102022015.png]]

"100 ns"가 유효한 데이터의 시작이다. 
- 유효한 주소의 시작이 "30 ns"여기서 "tACE" "70 ns"만큼 더했다.

![[Pasted image 20260102022041.png]]

"CEB"가 올라가고 "tHZCE" 시간 까지는 유효하게 데이터가 유지된다.
- 올라가면 RAM을 사용하지 않게 되겠군

![[Pasted image 20260102022115.png]]
