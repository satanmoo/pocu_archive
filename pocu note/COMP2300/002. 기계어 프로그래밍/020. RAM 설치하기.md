### RAM OEB 연결하기

![[Pasted image 20251231012656.png]]

"읽기/쓰기"를 토글로 구현한다고 가정하자. "OEB"는 `L`에서 `H`로 바꿔도 "읽기/쓰기"에서 변화를 주지 못한다. 띠라서 "OEB"를 언제나 `L`로 설정해도 괜찮지 않을까?

출력 비활성화는 우리가 다루지 않아도 된다.

결론적으로 우리 핀 구성에서는 "OEB"는 항상 `L`이다.

![[Pasted image 20251231012925.png]]

그래서 "OEB"에 `L`을 연결했다.

![[Pasted image 20251231013040.png]]

표에서 기존에 `X`(상관 없음)으로 표시한 값들이 `L`로 고정됬다.

### RAM WEB 연결하기

이제 "WEB"를 어떻게 할지 고민을 해보자.

"선택 해제/절전"에서 "WEB"는 아무 역할을 하지 못한다. 대신 "읽기/쓰기"를 바꿀 때 "WEB"에 어떤 값이 들어오는지가 결정적인 역할을 한다.

여기서 "CPU"에서 "읽기/쓰기"를 담당하는 핀을 떠올려보자.
- [[012. 핀 값 읽는 프로그램 작성하기#RWB]]

![[Pasted image 20251231013918.png]]

CPU의 "RWB"핀과 RAM의 "WEB"핀을 연결하면 되겠는데..?

그렇다면 이번에는 "NAND 게이트"가 필요할까?
- [[015. ROM 설치하기#NADN 게이트 SN74HC00]] 참고

"RWB"는 1일 때 READ, 0일 때 WRITE다.
"WEB"는 1일 때 READ, 0일 때 WRITE다.
그냥 NOT 연산 없이 그냥 연결해도 괜찮겠는데?

![[Pasted image 20251231014259.png]]

"RWB"와 "WEB"를 연결했다.

### RAM CEB 연결하기

![[Pasted image 20251231014402.png]]

"CEB"는 어떻게 해야하지?
- 0이 들어오면 "읽기/쓰기" 즉 활성화 상태
	- Chip Enable
- 1이 들어오면 절전 상태(비활성화)
