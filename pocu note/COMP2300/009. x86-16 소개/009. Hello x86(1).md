# Hello x86(1)

> [!info] 8088에서 콘솔 출력이 쉬워진 이유
> 6502에서는 콘솔(LCD 등)에 출력하려면 **제조사마다 다른 제어 방식**을 직접 구현해야 했다.
> LCD 제조사마다 제어 코드가 달랐기 때문에 붙이기도, 제어하기도 번거로웠다.
>
> 8088에서는 **ROM BIOS와 MS-DOS가 장치 드라이버를 제공**하기 때문에,
> 개발자는 하드웨어 제어 방식을 몰라도 된다.
>
> > [!tip] 개발자 입장에서 달라진 점
> > | | 6502 | 8088 (MS-DOS) |
> > |---|---|---|
> > | 콘솔 출력 | 제조사별 제어 코드 직접 구현 | BIOS/DOS 함수 호출로 끝 |
> > | 하드웨어 의존성 | 높음 | 낮음 (추상화됨) |
> > | 장치 교체 시 | 코드 수정 필요 | 드라이버만 교체 |

> [!info] `TITLE` — MASM 디렉티브
> `TITLE` 은 **어셈블리 명령어가 아닌 MASM 디렉티브**다.
> 프로그램 자체의 동작과는 무관하며, **어셈블러가 생성하는 목록(listing) 파일의 제목**을 정의한다.
>
> ```asm
> TITLE Hello x86   ; 목록 파일 상단에 "Hello x86"이 제목으로 표시됨
> ```
>
> > [!tip] MASM 레퍼런스 정의
> > **`TITLE text`** — Defines the program listing title.
