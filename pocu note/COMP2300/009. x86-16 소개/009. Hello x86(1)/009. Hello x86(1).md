# Hello x86(1)

> [!info] 8088에서 콘솔 출력이 쉬워진 이유
> 6502에서는 콘솔(LCD 등)에 출력하려면 **제조사마다 다른 제어 방식**을 직접 구현해야 했다.
> LCD 제조사마다 제어 코드가 달랐기 때문에 붙이기도, 제어하기도 번거로웠다.
>
> 8088에서는 **ROM BIOS와 MS-DOS가 장치 드라이버를 제공**하기 때문에,
> 개발자는 하드웨어 제어 방식을 몰라도 된다.
>
> > [!tip] 개발자 입장에서 달라진 점
> > | | 6502 | 8088 (MS-DOS) |
> > |---|---|---|
> > | 콘솔 출력 | 제조사별 제어 코드 직접 구현 | BIOS/DOS 함수 호출로 끝 |
> > | 하드웨어 의존성 | 높음 | 낮음 (추상화됨) |
> > | 장치 교체 시 | 코드 수정 필요 | 드라이버만 교체 |

> [!info] `TITLE` — MASM 디렉티브
> `TITLE` 은 **어셈블리 명령어가 아닌 MASM 디렉티브**다.
> 프로그램 자체의 동작과는 무관하며, **어셈블러가 생성하는 목록(listing) 파일의 제목**을 정의한다.
>
> ```asm
> TITLE Hello x86   ; 목록 파일 상단에 "Hello x86"이 제목으로 표시됨
> ```
>
> > [!tip] MASM 레퍼런스 정의
> > **`TITLE text`** — Defines the program listing title.
> 
> 목록 파일(program listing) 개념에 대서 굳이 알 필요가 없을 것 같다..

![[Pasted image 20260306193337.png]]

> [!info] `.DOSSEG` — MASM 디렉티브
> **MS-DOS 규약에 맞게 세그먼트 배치 순서를 자동으로 정렬**해주는 디렉티브다.
> 세그먼트를 어떤 순서로 메모리에 배치할지 MS-DOS 표준에 따라 결정된다.
>
> > [!tip] 실용적 관점
> > 내부 동작을 깊게 이해할 필요 없이, **MS-DOS용 MASM 프로그램을 작성할 때 항상 맨 위에 넣어주면 된다.**

![[Pasted image 20260306193545.png]]


> [!info] `.8086` / `.NO87` — CPU 대상 지정 디렉티브
> 이 어셈블리 코드가 **어떤 CPU에서 실행될 것인지** MASM에게 알려주는 디렉티브다.
> CPU 종류에 따라 MASM이 처리하는 방식이 달라지기 때문에 명시해줘야 한다.
>
> > [!note] `.8086` — 대상 CPU 지정
> > 8086 CPU를 대상으로 어셈블하겠다는 선언이다.
> > 수업에서는 **8088 CPU**를 사용하지만, 8088은 8086과 거의 동일하기 때문에
> > `.8086` 디렉티브로 커버가 된다. 애초에 `.8088` 은 존재하지 않는다.
>
> > [!note] `.NO87` — 코프로세서 비활성화
> > **8087**은 부동소수점 연산을 담당하는 **코프로세서(보조 CPU)** 다.
> > 수업에서는 코프로세서를 사용하지 않으므로 `.NO87` 로 명시적으로 비활성화한다.
> >

> [!info] `.MODEL TINY` — 메모리 모델 지정 디렉티브
> 세그먼트 메모리를 어떻게 관리할지 MASM에게 알려주는 디렉티브다.
> MODEL의 종류는 6가지가 있으며, 이 수업에서는 **TINY 모델**을 사용한다.
>
> > [!note] TINY 모델의 특징
> > 코드(`.CODE` ~ `END`)와 데이터(`.DATA`)가 **하나의 세그먼트(64KB 메모리 블록)** 안에 함께 들어간다.
> >
> > ```
> > [ 코드 + 데이터 ] ← 전부 하나의 세그먼트(64KB) 안에
> > ```
>
> > [!tip] TINY 모델 → `.COM` 파일 생성
> > TINY 모델을 사용하면 어셈블 결과물로 **`.COM` 파일**이 만들어진다.
> > `.COM` 파일은 코드와 데이터가 단일 세그먼트에 압축된 MS-DOS 실행 파일 형식이다.

![[Pasted image 20260306195203.png]]

> [!info] `.DATA` — 데이터 섹션
> 프로그램에서 사용할 **데이터를 선언하는 세그먼트**다.
> C언어의 데이터 섹션(`.data`)과 코드 섹션(`.text`)의 개념이 여기서 유래했다.
>
> ```asm
> .DATA
> msg DB "Hello, x86!$"
> ```
>
> > [!note] 각 요소 분석
> > | 요소 | 의미 |
> > |---|---|
> > | `msg` | 심볼(변수명). 6502의 데이터 블록 지시문과 유사한 개념 |
> > | `DB` | **Define Byte** - 뒤에 오는 데이터를 바이트 단위로 메모리에 배치 |
> > | `"Hello, x86!"` | 실제 문자열 데이터 |
> > | `$` | 문자열의 끝 표시 |
>
> > [!warning] 끝의 `$` — NULL 문자가 아니다
> > C언어의 `\0`(NULL)과 헷갈릴 수 있지만, 전혀 다른 개념이다.
> > `$`는 **MS-DOS가 문자열을 화면에 출력할 때 끝을 인식하기 위해 요구하는 종료 문자**다.
> > MS-DOS `int 21h` 출력 함수가 `$`를 만나면 출력을 멈춘다.
