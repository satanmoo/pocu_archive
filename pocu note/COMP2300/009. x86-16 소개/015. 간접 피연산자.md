# 간접 피연산자

## 간접 메모리 피연산자

![[Pasted image 20260310021522.png]]
> [!NOTE] 4개의 레지스터만 사용 가능
> 
> 앞에 `0000`이 붙어있는 레지스터만 사용 가능하다
>   - 베이스 레지스터: `bx`, `bp`
>   - 색인 레지스터: `si`, `di`

> [!attention] 레지스터 조합 규칙
> 
> 베이스만 or 색인만 or 베이스 + 색인 → 가능
> 
> 베이스 2개 or 색인 2개 → 불가능

> [!INFO] **간접**의 개념
>  
>  간접이기 때문에 레지스터에 담긴 주소 값(실행 중에 변할 수 있음)을 통해 메모리에 접근한다.
>  
>  바이너리 파일에는 "어떤 레지스터를 사용하라"는 정보가 피연산자로 인코딩된다.

## 4개의 간접 메모리 피연산자

![[Pasted image 20260310022725.png]]

### `mov bx, OFFSET table`

> [!quote] Programmer's Guide, Chapter 3 — The OFFSET Operator (p.60)
> An address constant is a special type of immediate operand that consists of an offset or segment value. The OFFSET operator returns the offset of a memory location.
> ```asm
> mov bx, OFFSET var    ; Load offset address
> ```

`OFFSET table` 이라는 표현이 어셈블 시점에 "table 변수의 주소"로 바이너리에 박힌다.
- 이 때문에 "a special type of immediate operand" 라고 표현한다.

> [!question] `mov bx table`  로 작성하면 어떻게 될까?
