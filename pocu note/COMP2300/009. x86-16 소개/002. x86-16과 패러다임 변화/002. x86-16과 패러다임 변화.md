# x86-16과 패러다임 변화
## intel 8088 CPU

![[Pasted image 20260303013356.png]]

> [!info] Intel 8086 vs 8088 비교
> 두 CPU는 거의 동일한 스펙이지만, **데이터 너비** 하나만 다르다.
>

### 8086과 2바이트 데이터 버스

> [!tip] 8086이 더 빠른 이유
> - 데이터 버스가 **16비트**이므로 한 번에 2바이트를 읽어올 수 있다
> - 8088은 **8비트** 버스라 16비트 데이터를 읽으려면 버스 접근을 **2번** 해야 함
> - 따라서 동일 클럭에서 **8086이 전반적으로 더 빠르다**

> [!example] 데이터 정렬(Alignment)로 8086 성능 극대화
> - 데이터를 **2바이트(짝수 주소) 단위로 align** 하면 8086이 **꽤 빨라진다**
> - 홀수 주소에 16비트 데이터가 걸쳐 있으면 8086도 버스 접근이 2번 필요해지기 때문
> - 즉, alignment는 8086에서 특히 중요한 최적화 포인트

> [!important] 데이터 핀 / 주소 핀이 핵심
> CPU 핀 구성에서 가장 중요하게 봐야 할 것은 **데이터 핀**과 **주소 핀**이다.
> 이 두 핀을 통해 메모리/장치와 실제 데이터를 주고받는다.

> [!warning] AD 핀 — 6502와 다르다!
> 6502에서는 주소 핀과 데이터 핀이 **완전히 분리**되어 있었지만,
> 8088/8086의 `AD` 핀은 **주소와 데이터를 시분할로 공유(Multiplexed)** 한다.
>
> - 어떤 시점에는 **주소 신호**가 지나가고
> - 어떤 시점에는 **데이터 신호**가 지나간다
>
> 즉, 같은 핀이 역할을 번갈아 가며 수행한다.

> [!tip] AD 핀의 현재 역할 확인 → 타이밍 다이어그램 필수
> AD 핀에 지금 **주소가 지나가는지, 데이터가 지나가는지**는
> **타이밍 다이어그램(Timing Diagram)** 을 봐야 알 수 있다.
> 단순히 핀 배열표만으로는 판단할 수 없다.

빨간색으로 하이라이트한 부분이 핀의 수에 따라 결정되는 **데이터 너비**(데이터 버스 너비)를 나타낸다.

## 8088을 먼저 배우는 이유

![[Pasted image 20260303020035.png]]

> [!question] 왜 8088을 먼저 배우는가?
> 
> **1. x86 명령어 빠른 습득**
> - 8088 명령어 → 현대 x86의 기초
> 
> **2. 세그먼트 메모리 이해**
> - 오늘날의 데이터 섹션/코드 섹션의 시작점
> - 이 개념이 현대까지 이어짐
> 
> **3. BIOS/OS 기능의 기초 동작법**
> - 하드웨어 직접 제어 ❌ → 추상화된 함수 호출 ✅
> - 하드웨어 제작과 프로그래밍의 분리
> 
> **4. 메모리 직접 접근 가능 (Real Mode)**
> - 8088 이후 세대에서는 이것도 불가능해짐
> - OS가 아직 힘이 약했던 과도기적 시대
> - → 메모리 보호 없이 자유롭게 접근 가능한 마지막 세대



## 복습 퀴즈

### 다음 중 8086과 8088의 차이점으로 옳지 않은 것은?

> [!question] Q. 다음 중 8086과 8088의 차이점으로 옳지 않은 것은?
>
> 1. 8086의 데이터 너비는 16비트, 8088의 데이터 너비는 8비트이다.
> 2. **8086은 주소 너비는 20비트, 8088의 주소 너비는 16비트이다.** ✅
> 3. 두 CPU 모두 x86-16 명령어 집합을 사용한다.
> 4. 두 CPU 모두 레지스터 크기가 16비트이다.
>
> > [!success]- 정답: 2번
> > 8086과 8088 모두 주소 너비는 **20비트**로 동일하다 (1MB 접근 가능).
> > 차이는 **데이터 버스 너비**뿐이다 (8086: 16비트, 8088: 8비트).
>
> > [!failure]- 오답 해설
> > **1번**: 맞는 설명. 8086은 16비트, 8088은 8비트 데이터 버스.
> >
> > **3번**: 맞는 설명. 둘 다 동일한 x86-16 명령어 집합을 사용한다.
> >
> > **4번**: 맞는 설명. 둘 다 내부 레지스터는 16비트이다.
