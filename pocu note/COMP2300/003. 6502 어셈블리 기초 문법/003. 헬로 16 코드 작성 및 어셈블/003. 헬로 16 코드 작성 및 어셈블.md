어셈블이 실패했다.

"Mnemonic"(니모닉)을 모른다는데..?

`clc`, `lda`, `adc` ,`sta` 이런 "opcode"라고 부른 것들이 니모닉에 해당한다.

에러 메시지를 읽어보면 `#$0A`, `#$06`, `$2200`을 니모닉으로 해석하는데..?

![[Pasted image 20260108001414.png]]

알고보니 "dasm"의 문법에서 "Mnemonic"은 한 번 들여쓰기 해야한다.

![[Pasted image 20260108001521.png]]

들여쓰기 하고 다시 어셈블하면 또 모르는 니모닉이라고 나온다.

"dasm"은 6502 cpu 니모닉만 지원하는게 아니다.
- 여러 [[018. 현대 CPU의 컴퓨터 추상화 계층#명령어 집합 구조(ISA)]]를 지원한다.
- ISA에 따라 니모닉도 달라진다.


![[Pasted image 20260108001750.png]]

`.PROCESSOR 6502`와 같이 `.`으로 시작하는 문을 지시문이라고 부른다

`.PROCESSOR 6502`는 6502 CPU에서 실행함을 어셈블러에게 지시하는 역할을 한다.

`.ORP $8000` ROM에 업로드 했을 때 시작 주소가 0x8000임을 어셈블러에게 알려준다.
- 우리가 만든 컴퓨터에서 ROM은 0x8000 주소 부터 시작하니까
- [[014. ROM 주소 매핑#CPU 주소 0x8000 이상일 경우]] 참고
- [[900. DASM 지시문]] 참고

> $nnn	hex

- 8000앞의 $는 hex를 나타낸다.

보충설명은 다음과 같다.

```asm
. PROCESSOR 6502
. ORG $8000        ; origin을 $8000으로 설정

clc               ; ← 이 명령어가 $8000에 생성됨
lda #$0A          ; ← $8001에 생성 (clc는 1바이트)
adc #$06          ; ← $8003에 생성 (lda #$0A는 2바이트)
sta $2200         ; ← $8005에 생성 (adc #$06는 2바이트)
```
