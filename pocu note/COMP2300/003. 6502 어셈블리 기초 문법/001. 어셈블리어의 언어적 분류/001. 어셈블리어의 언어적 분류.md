## 기계어 대신 어셈블리 언어를 사용하는 이유

![[Pasted image 20260105020246.png]]

어셈블리어를 기계어로 바꾸는 프로그램을 "어셈블러"라고 부른다. 

"어셈블러"는 여러 기능을 제공하는데, 이것이 변수명, 라벨, 매크로 등등이다
- 기본적으로 기게어는 모두 하드코딩

## 어셈블리어의 분류

![[Pasted image 20260105020700.png]]

어셈블리어는 언매니지드 언어다.

![[Pasted image 20260105020718.png]]

어셈블리어도 기계어에 비하면 고수준이다.

![[Pasted image 20260105020829.png]]

어셈블리어는 컴파일하지 않는다. 실행 파일이 나오긴 하는데...

![[Pasted image 20260105020850.png]]

"어셈블"이라는 과정을 통해서 실행 파일이 나오긴 한다.

![[Pasted image 20260105020942.png]]

어셈블리어는 타입이라는 개념이 없다.

![[Pasted image 20260105021011.png]]

절차적 언어는 명령을 순서대로 실행하는데, 이를 절차(Procedure)로 실행한다.

![[Pasted image 20260105021058.png]]

어셈블리어는 함수 같은 절차가 있는게 아니라, 명령만 존재한다. 명령형 언어다.
