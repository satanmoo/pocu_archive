### Implied Addressing

![[Pasted image 20260109015739.png]]

> These instructions act directly on one or more registers or flags internal to the CPU.
- 직접 적용한다..?
- 메모리 접근 없다라는 개념인듯

> Therefor, these instructions are principally single-byte instructions, lacking an explicit operand.
- 직접 적용하기 때문에 "single-byte instructions"이구나
- 즉 명시적인 피연산자 없이 니모닉만 있네

> The operand is implied, as it is already provided by the very instruction.
- "implied"
- 명령어 자체에서 피연산자가 암시되어 있다.

> Instructions targeting exclusively the contents of the accumulator may or may not be denoted by using an explicit "A" as the operand, depending on the flavor of syntax.
- `A`를 명시적으로 사용하면 배타적으로 누산기만 대상으로 하는군
- 근데 A를 사용하는 것은 선택이구나

### Immediate Addressing

![[Pasted image 20260109010906.png]]

>  Here, a literal operand is given immediately after the instruction.
- "literal operand"는 값 그 자체(상수)를 의미한다. 주소가 아니다.
- "immediately after the instruction" 명령어 뒤에 바로 붙어 있어서, "Immediate Addressing"이라고 부르는 구나

> The operand is always an 8-bit value and the total instruction length is always 2 bytes.
- 이건 스펙이다.

> In memory, the operand is a single byte following immediately after the instruction code.
- 윗 줄을 메모리 관점에서 한 번 더 설명했다.

> In assembler, the mode is usually indicated by a "#" prefix adjacent to the operand.
- 어셈블러에서 `#`접두사로 표시한다.

### Absolute Addressing

![[Pasted image 20260109010751.png]]

> Absolute addressing modes provides the 16-bit address of a memory location, the contents of which used as the operand to the instruction.
- 이것도 그냥 스펙
- 16비트 주소에 있는 내용이 피연산자로 사용된다.

> In machine language, the address is provided in two bytes immediately after the instruction (making these 3-byte instructions) in low-byte, high-byte order (LLHH) or little-endian.
- 기계어에서는 LLHH(리틀엔디언)
- 작은 쪽(하위 바이트)이 앞에 온다.

> In assembler, conventional numbers (HHLL order or big-endian words) are used to provide the address.
- 어셈블러에서는 HHLL(빅엔디언)
- 사람이 읽기 자연스러운("conventional") 순서


### Zero Page

![[Pasted image 20260116012139.png]]

> The 16-bit address space available to the 6502 is thought to consist of 256 "pages" of 256 memory locations each ($00…$FF). In this model the high-byte of an address gives the page number and the low-byte a location inside this page.

*16-bit* 주소를 256개 메모리 위치를 가진 256개의 페이지로 구성한다.
- 상위 비트는 페이지 번호
- 하위 비트는 페이지 내 위치

```text
$0000 ~ $00FF  →  Page 0  (Zero Page)
$0100 ~ $01FF  →  Page 1
$0200 ~ $02FF  →  Page 2
    ...
$FF00 ~ $FFFF  →  Page 255
```

> The very first of these pages, where the high-byte is zero (addresses $0000…$00FF), is somewhat special.

가장 처음 페이지(Zero-page)는 특별하다.

> The zero-page address mode is similar to absolute address mode, but these instructions use only a single byte for the operand, the low-byte, while the high-byte is assumed to be zero by definition.

*Zero-page* 주소 모드는 *절대 주소 모드*([[#Absolute Addressing]])와 유사하지만, 피연산자로 단일 바이트만 사용한다.
- 주소 상위 바이트(ADH)는 0으로 간주한다.

> Therefore, these instructions have a total length of just two bytes (one less than absolute mode) and take one CPU cycle less to execute, as there is one byte less to fetch.

피연산자로 단일 바이트만 사용하기 때문에 전체 명령어의 크기는 2바이트다.
- *절대 주소 모드*일 때 명령어의 크기보다 1바이트 작다.

*절대 주소 모드*보다 CPU 사이클도 하나 작다.
- 주소 상위 바이트를 *fetch* 하지 않아도 되기 때문이다.
