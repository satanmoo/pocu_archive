## A 모드

![[Pasted image 20260110014203.png]]

6502 CPU 기준 A레지스터에는 ALU의 계산 결과가 들어간다.
- 연산을 수행하는 부품이 ALU이다.
- 연산의 결과가 A레지스터에 들어간다.
- 따라서 니모닉의 기본 동작은 "A레지스터에 저장한다."라고 암묵적으로 동의하는 개념이다.

주소지정 모드의 정의를 만족하는가?
- 명령어가 피연산자를 찾는 방법을 정의하고 있냐?
- "피연산자로 A 레지스터를 사용한다" >> 만족

### 왜 가장 효율적인 주소지정 모드인가?

레지스터에 접근하는 것이 메모리에 접근하는 것 보다 빠르기 때문이다.
- CPU 내부에 레지스터가 존재한다.

### 실행 사이클?

CPU 클럭 사이클을 말한다.
- 사인파 1번

참고로 각 명령어의 주소 지정모드에 따른 세부 사항은 [여기](https://www.masswerk.at/6502/6502_instruction_set.html#modes)를 참고하자.

![[Pasted image 20260110015905.png]]

*ASL*도 A주소지정 모드가 가장 효율적이다.
- 명령어 크기도 작고
- 사이클도 작고

ASL = Arithmetic Shift Left
- 이진수면 값이 두배가 되겠군

lsr의 경우 이진수라면 값이 2로 나뉘겠군

ROR은 순환이다. Carry -> 비트0, 비트7 -> Carry

ROL도 순환이다. 비트0 -> Carry, Carry -> 비트7

### A 레지스터의 값을 어떻게 확인할까?

현재 확인할 수 있는 데이터는 데이터핀, 주소핀에서 나오는 데이터다.
- 스파이더를 꽂아서 확인했지?
- [[011. 스파이더 연결하기]]


A 레지스터는 CPU내부에 있어서 외부와 통신을 통해 확인할 수 있는 방법이 없다.

실행 사이클이 몇 사이클인지는 확인할 수 있다.
- 간접적으로 주소핀, 데이터핀으로 확인한다.
