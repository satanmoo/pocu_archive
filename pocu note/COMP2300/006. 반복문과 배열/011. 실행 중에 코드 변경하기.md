## RAM에 코드를 넣는 경우

![[Pasted image 20260221012153.png]]

> [!info] RAM 실행 흐름
> 1. **실행하기 전에 RAM으로 프로그램을 복사** — 외부 저장장치의 코드를 CPU에서 바로 실행할 수 없기 때문이다.
> 2. **RAM에 있는 프로그램을 실행** — 이 코드의 시작 주소(RAM 주소)를 읽어서 코드를 실행한다.

> [!note] 6502의 특징
> 만약 CPU 또는 운영체제가 위 시작 주소를 Read-Only로 보호하고 권한이 없다면 문제가 생긴다.
> - 6502 CPU는 하드웨어단에서 이런 제약이 없다.
> - 현재 강의에서 진행하는 6502 CPU는 운영체제도 없이 돌린다. 


## 예: 문자열 복사하기 (RAM 속 코드)

![[Pasted image 20260221012426.png]]

> [!NOTE] 가정
> 이미 RAM에 복사되어 코드가 존재한다고 가정한다.

![[Pasted image 20260221012701.png]]

> [!tip] 핵심 아이디어
> `src`와 `dst`는 루프가 돌 때마다 계속 값이 바뀐다 → 그래서 `??`로 표기한다.
>
> ```asm
> inc loop+1   ; lda src 명령어의 피연산자(src 주소)를 1 증가
> inc store+1  ; sta dst 명령어의 피연산자(dst 주소)를 1 증가
> ```
> `loop+1`, `store+1`은 각 명령어의 피연산자가 저장된 메모리 주소다. ([[005. 6502 명령어의 구성]] 참고)

### 메모리까지 확인해보자

![[Pasted image 20260221013754.png]]

> [!info] 초기 메모리 배치 가정
> - 페이지 0(`$00~`)에 프로그램 코드가 들어가 있다.
> - `$70~`에 원본 문자열(src)이 들어가 있다.
> - `$90~`이 복사 대상(dst) 영역이다.

![[Pasted image 20260221013904.png]]

> [!example] 초기 상태 (루프 시작 전)
> - `LEN=11`, `src=$70`, `dst=$90`
> - `loop`: `lda src` → 기계어 `A5 ??` (`??`=`$70` 이 들어갈 자리)
> - `store`: `sta dst` → 기계어 `85 ??` (`??`=`$90` 이 들어갈 자리)
> - `$70`~`$7A`: "Hello World" 바이트 저장됨
> - `$90`: 아직 비어 있음

![[Pasted image 20260221013926.png]]

> [!example] `sta dst` 실행 후
> - `A` ← `$70`의 첫 번째 바이트(`H` = `$48`) 로드


![[Pasted image 20260221013943.png]]

> [!example] `inc loop+1` 실행 후
> 
> - `lda` 피연산자 `$70` → `$71`


![[Pasted image 20260221014036.png]]

> [!example] `inc store+1`
> - `sta` 피연산자 `$90` → `$91`


![[Pasted image 20260221014048.png]]

> [!example] `lda store+1`
> - 현재 `dst` 주소(`$91`)를 A에 로드


![[Pasted image 20260221014110.png]]

> [!example] `cmp \#dst+LEN`
> - `$90 + 11 = $9B`와 비교 (dst의 마지막)



> [!note] DASM 어셈블 시 상수 결정
> `#dst+LEN` 같은 표현은 **어셈블 도중에 상수 주소로 바뀐다.** 즉, 정적으로 결정된다.
> DASM 어셈블러가 `dst`의 주소 + `LEN`을 계산해 피연산자 위치의 메모리 주소를 확정한다.

![[Pasted image 20260221014215.png]]

> [!example] `bne loop`
> - 아직 `store+1` 이 `$9B`에 도달하지 않았으면 계속 반복

> [!warning] 이 방식의 단점
> - **느리다** — `inc loop+1`, `inc store+1`, `lda store+1`, `cmp` 등 추가 명령어가 많다.
> - **코드 가독성이 떨어진다** — 피연산자를 직접 수정하는 방식은 이해하기 어렵다.
