## 예: 다시 보는 배열의 합 구하기

![[Pasted image 20260218161011.png]]

### 비효율적으로 배열의 요소를 앞에서 부터 순회해 더하는 코드

```asm
; 배열의 합을 구하는 코드  
    .PROCESSOR 6502  
    .ORG $8000  
  
LENGTH=4  
  
sum=$1000   ; 출력 변수  
; --- 스택 초기화  
    ldx #$FF  
    txs  
  
; --- 배열의 요소를 순회해서 누적하는 코드  
    clc  
    lda nums  
    ldx #1  
.loop:  
    cpx #LENGTH  
    beq .save  
    adc nums,X  
    inx  
    jmp .loop  
; --- 누적한 결과를 출력 변수에 저장  
.save:  
    sta sum  
  
    .ORG $C000,0  
nums:  
    .BYTE $00,$01,$02,$03  
  
    .ORG $FFFC,0  
    .WORD $8000  
    .WORD $0000
```

### 강의에서 사용한 효율적으로 배열의 요소를 뒤에서 부터 순회해 더하는 코드

![[Pasted image 20260218213323.png]]

![[Pasted image 20260218213558.png]]

> LEN=4

문자열 처럼 널문자로 배열의 끝을 알 수 있는 상황이 아니기 때문에, 배열의 길이를 선언했다.

![[Pasted image 20260218214153.png]]

> clc

덧셈 하기 전 *캐리 플래그* 지우기

![[Pasted image 20260218214217.png]]

> lda \#0
> ldx \#LEN-1

`LDA #0`은 누산기 A를 0으로 초기화한다.
- 이 값이 지금부터 배열 합을 누적할 시작점이므로, 이전 값이 남아 있으면 합이 틀어지기 때문이다.

`#LEN-1`의 `-1`은 **DASM이 제공하는 어셈블 타임(compile-time) 상수 표현식 문법**이다.

실행 중에 빼는 것이 아니라, **어셈블러가 기계어로 변환할 때 미리 계산**한다. `#` 뒤에 단순 숫자뿐 아니라 상수 산술 표현식도 쓸 수 있는 것이 DASM의 기능이다.
- `#LEN-1`은 DASM이 어셈블 시점에 `4-1 = 3`으로 계산해 `#3`과 동일하게 처리

![[Pasted image 20260218220047.png]]

> bmi

`bmi`는 **Branch if MInus**의 약자로, **N(Negative) 플래그가 1일 때 분기**하는 명령어다.

이 코드에서는 `ldx #LEN-1`로 X를 배열의 마지막 인덱스(3)부터 시작해 `dex`로 하나씩 줄여나간다.

X가 0인 상태에서 `dex`를 한 번 더 실행하면 X = `$FF` (255)가 되는데, 6502는 이를 **부호 있는 정수 -1**로 해석해 N 플래그를 세운다. `bmi`는 이 시점을 포착해 루프를 탈출한다.

앞에서부터 순회하는 코드는 `cpx #LENGTH` + `beq` **두 명령어**로 루프 종료를 판단했지만, 뒤에서부터 순회하면 `dex`가 N 플래그를 자동으로 설정하므로 `bmi` **하나만으로** 처리할 수 있어 더 효율적이다.

![[Pasted image 20260218221032.png]]

> adc nums,x

`adc nums,X`는 **Add with Carry**로, **`A = A + nums[X] + C`** 를 수행한다.

- `nums`는 배열의 시작 주소(베이스 주소)이며, X 레지스터가 **인덱스** 역할을 한다.
- 즉, `nums,X`는 `nums + X` 번지의 메모리 값을 읽는 **절대 인덱스 어드레싱(Absolute Indexed Addressing)** 이다.
- C(캐리 플래그)가 더해지기 때문에, 루프 시작 전 반드시 `clc`로 캐리를 초기화해야 첫 덧셈이 올바르게 동작한다.

이 코드에서는 X가 `LEN-1`(3)에서 0까지 감소하며, 매 반복마다 `nums[3]`, `nums[2]`, `nums[1]`, `nums[0]` 순으로 A에 누적한다.

![[Pasted image 20260218221200.png]]

> dex

`dex`는 **Decrement X**로, **X 레지스터를 1 감소**시키는 명령어다 (`X = X - 1`).

- 감소 결과에 따라 **N(Negative) 플래그**와 **Z(Zero) 플래그**를 자동으로 갱신한다.
- 이 코드에서는 배열 인덱스를 뒤에서 앞으로 줄여나가는 역할을 하며, X가 0에서 한 번 더 감소해 `$FF`(-1)가 되는 순간 N 플래그가 세워져 `bmi`가 루프를 탈출한다.

덕분에 별도의 비교 명령어(`cpx`) 없이 `dex` → `bmi` 두 명령어만으로 루프 종료를 처리할 수 있다.

![[Pasted image 20260218221331.png]]

> jmp loop

`jmp loop`는 **Jump**로, **무조건 `loop` 레이블로 분기**하는 명령어다.

- 조건 없이 항상 지정한 주소로 이동하며, C의 `goto`와 동일한 동작이다.
- 이 코드에서는 `bmi`가 분기하지 않은 경우(아직 루프를 더 돌아야 할 때) 다시 루프 시작으로 돌아가기 위해 사용된다.

> [!INFO] 
> `bmi`(조건 분기)와 `jmp`(무조건 분기)가 짝을 이뤄 루프를 구성한다.
> - **N 플래그 = 1** → `bmi`가 루프 탈출
> - **N 플래그 = 0** → `bmi` 무시 → `jmp loop`로 다음 반복

![[Pasted image 20260218221653.png]]

첫 번째 요소까지 다 더하고 루프로 돌아올 때 *X 레지스터* 의 값은 `$FF` 이 된다.
- *N 플래그* 가 세워진다.
- `exit` 으로 점프한다.

![[Pasted image 20260218221810.png]]

> sta sum

`sta sum`은 **Store Accumulator**로, **누산기 A의 값을 `sum` 주소에 저장**하는 명령어다 (`sum = A`).

- `sum`은 코드 상단에 `sum=$1000`으로 선언된 **출력 변수**다.
- 루프가 끝난 뒤 A에 누적된 배열의 합을 메모리에 기록하는, 프로그램의 마지막 단계다.
- `adc`는 결과를 A에만 임시로 보관하므로, 최종적으로 `sta`를 통해 메모리에 써야 값이 보존된다.

## 배열의 합 구하는 코드 비교

![[Pasted image 20260218222006.png]]

>[!INFO] 반복문의 경우 네번의 반복을 돌렸기 때문에 N의 값은 4로 61 사이클이 걸린다.

반복문을 사용하면 보통 사이클 수는 늘어나지만, 바이트(코드의 크기)는 줄어든다.

따라서 반복 횟수를 알고 있는 경우(배열의 길이를 알고 있는 경우) 반복물을 쓰지 않는게 사이클 측면에서 유리하다.
- 고수준 컴파일러의 unloop 최적화

하지만 실행 중에 데이터 크기가 결정되는 경우 반복문을 쓸 수 밖에 없다.

> [!TIP] 컴파일 타임에 데이터 크기를 이미 아는 경우 *unloop* 가 성능적으로 유리하다.
