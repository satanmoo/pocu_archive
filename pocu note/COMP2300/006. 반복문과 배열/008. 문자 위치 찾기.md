## 예: 문자 위치 찾기

![[Pasted image 20260220005713.png]]

### 직접 작성한 문자 위치 찾기 코드

```asm
; 문자열에서 특정 문자의 위치를 찾는 코드  
    .PROCESSOR 6502  
    .ORG $8000  
  
index=$1000   ; 출력 변수  
target=$1001  ; 찾을 문자  
  
;--- 찾을 문자를 메모리에 저장  
    lda #'L    ; 찾을 문자 'L'의 ASCII 코드 로드  
    sta target ; 메모리에 저장  
  
;--- 스택 초기화  
    ldx #$FF  
    txs  
;--- 반복문 전 초기화  
    ldx #0       ; 문자열의 시작 인덱스 설정  
  
;--- 문자열을 순회하면서 특정 문자를 찾는 반복문  
.loop:  
    lda str,x      ; 문자열에서 현재 인덱스의 문자 로드  
    beq .not_found  ; 널 문자(0)이면 찾지 못함  
    cmp target     ; 현재 문자가 찾을 문자와 같은지 비교  
    beq .found     ; 같으면 위치 저장  
    inx            ; 인덱스 증가  
    jmp .loop       ; 반복  
  
.found:  
    stx index      ; 찾은 위치를 출력 변수에 저장  
    jmp .end        ; 프로그램 종료  
  
.not_found:  
    lda #$FF       ; 찾지 못한 경우 -1(0xFF)을 저장  
    sta index  
  
.end:  
    jmp .end         ; 프로그램 종료  
; --- 문자열 데이터 (레이블로 주소 지정)  
str:  
    .BYTE "HELLO WORLD"  
    .BYTE 0      ; 문자열의 끝을 나타내는 널 문자  
  
    .ORG $FFFC,0  
    .WORD $8000  
    .WORD $0000
```

### 수업에서 다룬 문자 위치 찾기 코드

![[Pasted image 20260220013312.png]]

> str:
> 	.BYTE "Hello World"
> estr:

>  LEN=estr-str

> [!NOTE] `.BYTE` 지시문을 응용해 두 레이블의 주소 차이로 문자열의 길이를 구했다.

> [!TIP] 널 문자를 활용하는 방법이 더 좋다

> [!INFO] *X 레지스터* 를 0으로 초기화해 문자열의 앞(0번 색인)부터 순회한다.
> 뒤에서 순회를 시작할 수 없는 이유는, 요구 사항 자체가 앞에서부터 검색해 **첫 번째** 색인을 반환하는 것이기 때문이다.

![[Pasted image 20260220013924.png]]

> cmp str,x

> [!NOTE] *A 레지스터* 에 찾을 문자가 이미 저장되어 있기 때문에 `cmp str,x` 로 바로 비교할 수 있다.

![[Pasted image 20260220014049.png]]

> notfound:
> 	ldx \#$FF

> [!TIP] `goto` 를 사용해 조건 비교 없이 바로 점프할 수 있다.
